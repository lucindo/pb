### BLOCKER — CR-01 residual: resonant settings read/write split-brain

- **Write:** `persistedSetSettings` → `saveResonantSettings(next)` → `practices.resonant.settings` (App.tsx:416)
- **Read on mount:** `loadSettings()` → `coerceSettings(readEnvelope().settings)` → flat `env.settings` (App.tsx:110, settings.ts:45)
- `env.settings` is **never written** after Phase 31 — `saveSettings()` is dead production code (test-file usages only).
- **User impact:** resonant BPM/ratio/duration/mode revert on every reload. Fresh v2 users revert to defaults; v1-migrated users revert to their migration-time values.
- **Why missed:** `App.persistence.test.tsx` seeds only `version: 1` envelopes; no test seeds a v2 envelope with `practices.resonant.settings` populated and remounts to verify the read path.
- **Fix:** App.tsx:110 — seed `initialSettings` from `loadPractices().resonant.settings` (the value is already available at App.tsx:115 as `initialPractices.resonant.settings`). One-line change; `loadSettings`/`saveSettings` then become removable dead code.
