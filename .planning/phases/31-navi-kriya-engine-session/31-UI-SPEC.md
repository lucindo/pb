## Component Inventory

New components and modified files for Phase 31. Executor follows the patterns

### 1. NK Shape â€” count-in-shape (new rendering branch)

**Pattern:** Extend existing shape components with an `nkCount` prop (or create
a thin `NKShape.tsx` wrapper â€” RESEARCH OQ-2 recommends the wrapper). The
recommended approach is an `NKShape.tsx` wrapper that renders the user's chosen
variant shape (`OrbShape` / `SquareShape` / `DiamondShape`) in NK mode.

**Visual spec:**
- Shape renders at MID_SCALE resting state (not breathing scale).
- On each OM: CSS animation restarts via `key={nk-${nkCount}}` prop change â€”
  scale pulses up to MAX_SCALE then settles back to MID_SCALE. Duration: 200ms
  ease-out up, 300ms ease-out settle.
- `prefers-reduced-motion: reduce` â€” shape stays locked at MID_SCALE, no
  pulse animation. The count number still updates. (Matches Phase 2 D-05/06/07
  static fallback principle.)
- The `CueGlyph` slot is replaced by the live count number. The count number:
  - Class: `text-7xl sm:text-8xl font-semibold tracking-tight`
  - Color: `text-[var(--color-orb-in-text)]` (same as OrbLeadIn digit)
  - Content: plain integer, e.g. `47` (not `47/100`) â€” D-02 locked.
- During the settle settle period (before first OM), the shape renders at
  MID_SCALE with count `0`. No digit shown during the settle delay; once the
  front marker fires and the engine starts, count begins at 1.
- The shape does NOT crossfade orb-in/orb-out layers during NK (no `data-phase`
  attribute flipping). It holds `orb-layer--in` throughout the session.
- No expanding ring animation â€” D-04 locked.
- `data-variant` attribute: set to the user's chosen variant (`orb` / `square`
  / `diamond`) â€” same as existing BreathingShape pattern.
- `aria-label`: `"Navi Kriya session: OM ${count}, phase ${phaseLabel}"` â€”
  updates on each OM for screen readers.

**CSS animation for pulse (new, NK-only):**
```css
@keyframes nk-om-pulse {
  0%   { transform: translate3d(0,0,0) scale(var(--orb-scale-mid)); }
  30%  { transform: translate3d(0,0,0) scale(var(--orb-scale-max)); }
  100% { transform: translate3d(0,0,0) scale(var(--orb-scale-mid)); }
}
.nk-om-pulse {
  animation: nk-om-pulse 500ms ease-out forwards;
}
@media (prefers-reduced-motion: reduce) {
  .nk-om-pulse {
    animation: none;
  }
}
```
The `key` prop change on the shape wrapper forces React to remount the animated
element, restarting the animation on every OM.

**Pattern:** Mirrors `SessionReadout`'s outer container styling exactly.

**Visual spec:**

- Inner: `mt-4 flex items-stretch rounded-[1.25rem] bg-[var(--color-breathing-surface)]/80 px-2 py-3 text-[var(--color-breathing-accent-strong)]` â€” byte-identical to the stretch readout row.
- Three cells, separated by `<span className="w-px self-stretch bg-[var(--color-breathing-muted)]/40" />`:

  | Cell | Label | Value |
  |------|-------|-------|
  | Phase | `strings.nkReadout.phaseLabel` â†’ "Phase" | `strings.nkReadout.front` â†’ "Front" or `strings.nkReadout.back` â†’ "Back" |
  | Round | `strings.nkReadout.roundLabel` â†’ "Round" | e.g. `"2 / 3"` (current round / total rounds) |
  | Count | `strings.nkReadout.countLabel` â†’ "Count" | e.g. `"100"` (target for current phase) |

- Cell label class: `text-[0.65rem] font-semibold uppercase tracking-[0.16em] text-[var(--color-breathing-muted)]` â€” same as stretch readout.
- Cell value class: `text-lg font-semibold` â€” same as stretch readout.
- `aria-label` on container `<section>`: `strings.nkReadout.readoutAriaLabel` â†’ `"Navi Kriya session readout"`.

**Width constraint:** "Front" and "Back" cell labels must not exceed the width

## Interaction Contract

### NK session start flow

1. User taps "Start session" in NK controls area.
2. `AudioContext` created synchronously in `onNKStartClick` (user gesture chain â€” browser autoplay policy).
3. Screen transitions to NK session view: NKShape at MID_SCALE (count `0`), NKSessionReadout strip visible (Phase: Front, Round: 1 / N, Count: frontCount), SessionControls visible.
4. Settle delay (~3â€“5 s, adjustable constant): shape is visible but no count increment. Shape holds MID_SCALE.
5. Front marker cue fires. LEAD_MS (~700ms) delay.
6. Engine starts: first OM counted. Shape pulses. Count increments to 1.
7. Per-OM: count increments, shape pulses (or static under reduced-motion).
8. Frontâ†’Back transition: back marker cue fires, readout flips to "Back", count resets to 0 + 1.
9. Round completion â†’ next round: front marker, round number increments.
10. Practice complete: end chord cue fires, NK completion dialog opens.

### NK pause / resume

- Pause: `SessionControls` "End session" / the same control area. Recommend: pause is triggered by `SessionControls`' primary button (which shows "End session" when running â€” existing posture). Or add a dedicated Pause affordance alongside End. Claude's discretion â€” see CONTEXT.md.
- Paused: count dimmed, shape static at MID_SCALE, `SessionControls` shows the start-alike button for Resume.
- Resume: tapping the Resume / "Start session" button reschedules the next OM.
- End early: shows `EndSessionDialog` for confirmation.

### NK settings interaction

- All four steppers (`SettingsStepper`) are disabled while a session is running (consistent with resonant pattern).
- Per-OM cue toggle can be changed while running (engine reads `eng.current.cueOn`).
- Duration estimate updates live as any value changes (`aria-live="polite"`).
- `frontCount` stepper must only offer multiples of 4 â€” domain-enforced by `coerceNaviKriyaSettings`. The stepper options array must be pre-filtered to multiples of 4.

### Focus management

- NK completion dialog: initial focus on the "Close" button (safe default â€” not destructive). Follows `EndSessionDialog` focus pattern.
- Early end dialog: initial focus on "Keep going" (cancel) â€” D-12 locked, existing `EndSessionDialog` behavior.

### Reduced-motion

- NK shape pulse: static at MID_SCALE. Count number still updates.
- Dialog fade: disabled (existing `@media prefers-reduced-motion` rule in `theme.css` sets `dialog.modal-fade { transition: none !important }`).
- All transition classes: `motion-reduce:transition-none` already present on existing button classes â€” carry forward for NK buttons.
