<objective>
Synthesize the four Navi Kriya cue sounds (NK-05 + NK-06's tick) as a new
standalone audio module `src/audio/nkCueSynth.ts`. Each cue is a pure Web Audio
builder that takes `(audioCtx, when, destination, timbre)`, returns a `CueHandle`,
and reads `TIMBRE_PRESETS[timbre]` so the cue honors the user's chosen timbre
(D-05). Front marker = rising two-tone, back marker = falling two-tone (D-06),
per-OM tick = soft barely-there short tone (D-07), end chord = resolved low
multi-note chord that rings out (D-08).

Purpose: the NK session needs audible markers and a per-OM tick. This module is
the audio half of the engine — App.tsx (Wave 2) injects these as the engine's
`NKAudioCallbacks`.

Output: `src/audio/nkCueSynth.ts` (new) + `src/audio/nkCueSynth.test.ts` (new).
