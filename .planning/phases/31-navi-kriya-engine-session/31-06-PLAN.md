<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user gesture -> AudioContext | the Start tap is the only place an `AudioContext` may be constructed (browser autoplay policy) |
| localStorage -> NK settings/stats | `nkSettings` loaded at mount, `naviKriya.stats` written on session end — read back through coercers |
| timer callbacks -> React tree | settle/lead/OM `setTimeout`s mutate refs and state; must be cancelled on unmount/end |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-13 | Denial of Service | Settle/lead/OM timers firing into a torn-down `App` (reload, PWA dismissed) mid-session | mitigate | Settle/lead timer ids and the engine timer are held in refs; `onNKEndClick`/cleanup call `clearTimeout` on all of them; `useNKEngine`'s `useEffect` cleanup clears its own timer (RESEARCH Pitfall 4). |
| T-31-14 | Tampering | CR-01 regression — a stray `saveSettings()` resonant write persisting to the wrong location | mitigate | `persistedSetSettings` switched to `saveResonantSettings`; `App.persistence.test.tsx` asserts the new write path; grep gate confirms no resonant-settings `saveSettings(` write remains (RESEARCH Pitfall 5). |
| T-31-15 | Tampering | An NK session corrupting Resonant Breathing stats | mitigate | `recordNaviKriyaSession` writes only the `naviKriya` slice (plan 03 guarantee); `App.session.test.tsx` asserts resonant stats unchanged after an NK session (NK-08). |
| T-31-16 | Repudiation | An AudioContext-construction failure silently leaving the session with no audio | accept | Per UI-SPEC error-state section, audio failure is a silent degradation (cues just do not fire) — consistent with the existing `audioEngine` behavior; no new error UI is in scope. |
