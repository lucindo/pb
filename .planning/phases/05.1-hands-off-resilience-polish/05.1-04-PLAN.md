Box-model analysis (deeper root cause uncovered by 05.1-UAT.md Task 3):
- Container: `<div className="relative mx-auto my-12 grid place-items-center" style={{ width: 'var(--orb-size)', height: 'var(--orb-size)' }}>` — grid layout with `place-items-center`.
- Three children of this container are `position: absolute`: outer-ring span, inner-ring span, `.orb` div.
- Per CSS spec, absolutely-positioned children of a grid container have ambiguous sizing when only `inset-0` (`top/right/bottom/left: 0`) is specified — the resolution depends on the implicit grid track that the abs-pos child is placed in (auto track), and Safari Desktop sizes that track to the largest non-abs sibling.
- The largest non-abs sibling here is the inner ring (which IS sized via explicit `width: 58%; height: 58%` per WR-03), so Safari Desktop sizes the auto-grid-track to 58% and then resolves `inset-0` against that 58% box → `.orb` collapses to 209 px (= 0.58 × 360 px).
- The inner ring dodges this because it does NOT use `inset-0`; it uses explicit `left:50%; top:50%; -translate-{x,y}-1/2; width:58%; height:58%` which sizes the box independently of the implicit track.
- The outer ring's `inset: -1.5px` shorthand happens to resolve correctly on Safari Desktop because the negative offsets force a non-zero size against a known reference (the four edges of the abs-pos containing block resolved to the container's padding-box) rather than depending on the auto-track sizing — Plan 02's measurement of 363 px confirms this.
- BUT: Plan 04 unifies all three abs-pos children to the SAME explicit-edge form (D-21) so future refactors don't accidentally re-introduce a `inset` shorthand that lands on the broken track-sizing path.

Fix per D-19/D-20/D-21:
- `.orb` div in both render sites: drop `inset-0` Tailwind class, add inline `style={{ ...existingTransform, left: 0, top: 0, width: '100%', height: '100%' }}` (merge with existing `transform` style).
- Outer-ring span in both render sites: replace `style={{ inset: '-1.5px' }}` with `style={{ left: '-1.5px', top: '-1.5px', right: '-1.5px', bottom: '-1.5px' }}`.
- Inner-ring span: untouched (D-26 — already uses WR-03 pattern).
- `.orb-layer--in` / `.orb-layer--out` children of `.orb`: untouched per D-23 default (b) — verify via grep + tsc + vitest after parent fix lands; if `.orb` post-fix measures correctly the layers' own `inset-0` should resolve correctly because their containing block is now the explicitly-sized `.orb` (not a grid auto-track).
