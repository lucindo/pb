## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] vi.spyOn(FakeAC.prototype, 'resume') pattern from plan spec fails with FakeAudioContext**
- **Found during:** Task 4 (adding resume-path tests)
- **Issue:** The plan spec used `vi.spyOn(FakeAC.prototype, 'resume')` to intercept resume calls, but FakeAudioContext declares `resume = vi.fn(async () => {...})` as an arrow-function class field (own instance property). Prototype spying only works for methods defined on the prototype, not instance fields. Running the tests as written in the plan produced: `Error: The property "resume" is not defined on the object.`
- **Fix:** Introduced `SpyableAC` class inside the test `describe` block that declares `resume` as a prototype method (not an arrow-function field). `vi.spyOn(SpyableAC.prototype, 'resume')` intercepts correctly. Each test stubs `window.AudioContext` with `SpyableAC` via `vi.stubGlobal`. The `afterEach` `vi.unstubAllGlobals()` restores the original FakeAudioContext for other tests.
- **Files modified:** `src/hooks/useAudioCues.test.tsx`
- **Verification:** All 15 tests pass (11 original + 4 new); full suite 280/280 green
- **Committed in:** `cced07b` (Task 4 commit)
