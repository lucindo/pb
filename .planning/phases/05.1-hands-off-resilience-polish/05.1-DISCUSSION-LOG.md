## Audio Resume Site

| Option | Description | Selected |
|--------|-------------|----------|
| Inside `useAudioCues` hook | Mirror useWakeLock D-03/D-04 structure: hook owns its own visibility listener, ref-gated guard, mount/unmount cleanup. Symmetry with Phase 5 makes the pattern obvious; engineRef + activeCues live in the hook anyway. | ✓ |
| Inside `audioEngine` module | Engine owns the listener directly on its AudioContext. Couples lifecycle to the engine itself. Loses React-cleanup symmetry; engine is currently zero-React/zero-DOM-event — introducing a document listener there breaks that boundary. | |
| Inline in App.tsx | Add a useEffect in App.tsx that listens to visibilitychange and calls audio.resume() when appPhase==='running'. Lightest in lines of code, but App.tsx is already the largest file and Phase 5 D-03 chose hook-ownership for exactly this reason. | |

**User's choice:** Inside `useAudioCues` hook (Recommended)
**Notes:** Captured as D-01 / D-02. Drives the "engine module stays React-free" boundary forward.

## Audio Resume Gating + Make-Up Cue

### Question 1 — Gating

| Option | Description | Selected |
|--------|-------------|----------|
| Whenever an engine exists | Gate on `engineRef.current !== null`, mirroring useWakeLock's `wasAcquiredRef` gate. Covers running AND lead-in. Simplest invariant: if AC was constructed, attempt to resume. Matches D-04 ref-gated re-acquire posture. | ✓ |
| Only when status === 'lead-in' or appPhase === 'running' | Tighter gate. Hook would need to know about appPhase, which lives in App.tsx. Cross-boundary coupling — hook starts depending on caller state instead of its own refs. | |
| Whenever audioCtx.state === 'suspended' | Read AC state directly. Reactive but iOS may report wrong state during display sleep window; relies on browser truthfulness and adds polling-feel. | |

**User's choice:** Whenever an engine exists (Recommended)
**Notes:** Captured as D-03 / D-04.

### Question 2 — Make-up cue

| Option | Description | Selected |
|--------|-------------|----------|
| Wait for next phase boundary | Mirrors Phase 3 D-08 unmute-waits-for-boundary rule. The next In/Out cue fires from the existing scheduleNextCue dispatch. Clean, no make-up scheduling code, no audible 'late strike' that would feel like a glitch. | ✓ |

**User's choice:** Wait for next phase boundary (Recommended)
**Notes:** Captured as D-05 / D-06. Preserves Phase 3 dual-anchor invariant.

## Orb Scale Fix Approach

### Question 1 — Fix mechanism

| Option | Description | Selected |
|--------|-------------|----------|
| Bump MAX_SCALE buffer (e.g. 1.02) | Smallest delta. Change MIN_SCALE/MAX_SCALE/MID_SCALE constants + matching CSS tokens. Risk: visual shift on Chromium where the gap doesn't exist (orb would now overflow ring slightly). Also affects MID_SCALE via the average formula. | |
| Compensate outer ring sizing | Outer ring uses Tailwind `inset-0` + `border-solid` (default `box-sizing: border-box`). Switch outer ring to explicit width/height: 100% with `box-sizing: content-box` OR negative `inset` of -1.5px to push the ring's visual edge out by its own border-width. Keeps the orb math untouched. Matches the inner-ring's explicit sizing pattern. | ✓ |
| Replace ring with SVG circle | Render outer ring as an SVG `<circle>` with `r=50%` and stroke-width — SVG centerline-stroke math is fully deterministic across browsers. Larger refactor; affects two render sites (body + lead-in). | |

**User's choice:** Compensate outer ring sizing (Recommended)
**Notes:** Captured as D-10 / D-12 / D-13. Leaves orb scale constants and reduced-motion contract untouched.

### Question 2 — Inner ring also?

| Option | Description | Selected |
|--------|-------------|----------|
| No — only outer ring needs fix | UAT finding only flagged outer-ring gap at peak inhale. Inner ring is already explicitly sized via width/height percent (BreathingShape.tsx:80-83) so it doesn't share the inset-0 + border-box arithmetic. Don't fix what isn't broken. | ✓ |
| Yes — apply same compensation to both | Symmetric box-model treatment. Cleaner mental model but expands the scope and may shift inner ring rendering on Chromium where it currently looks correct. | |

**User's choice:** No — only outer ring needs fix (Recommended)
**Notes:** Captured as D-11.
