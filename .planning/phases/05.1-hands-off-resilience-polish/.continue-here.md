# BLOCKING CONSTRAINTS — Read Before Anything Else

> Each constraint below was discovered through actual UAT failure during this session. Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: **Plan 01 audio-resume spec underestimated iOS Safari auto-close behavior** — `audioCtx.resume()` is empirically insufficient on real iPhone iOS Safari after a lock/unlock cycle (even <30s). The spec assumed iOS only suspends the AC; in practice iOS likely closes it, so resume() rejects and silent-absorbs (D-09) with no recovery. Mute/unmute toggle does NOT restore audio either. Only starting a new session works (because that constructs a new AudioContext). Structural mitigation: BEFORE writing or accepting any new spec for the resume path, run the diagnostic in `human_actions_pending` first to confirm whether audioCtx.state is 'suspended' or 'closed' on real iOS. Do not take the spec's word over the device's behavior.

- [ ] CONSTRAINT: **Plan 04's first ship was insufficient — Safari Desktop required 5 hotfixes after the initial fix** — initial fix used `width:100%; height:100%; left:0; top:0` which sized the orb box correctly (360 px) but FROZE Safari's CSS transition (computed transform stuck at scale(0.58) despite inline transform updating to scale(0.99)). Root cause: Safari restarts a CSS transition on every inline-style reassignment when sizing properties are present alongside transform. Mitigation cascade required: (1) swap to 4-edge anchoring (left:0/right:0/top:0/bottom:0, no width/height), (2) drop the `.orb { transition: transform 200ms }` from theme.css entirely (rAF cadence at 60fps is enough smoothing), (3) add `will-change: transform` for Firefox compositor promotion, (4) inline `translate3d(0,0,0) scale(s)` for further Firefox GPU promotion. Structural mitigation: any future plan that introduces or modifies CSS transitions on elements receiving per-frame inline-style updates MUST verify on real Safari Desktop, NOT just jsdom or auto-promoting browsers.

- [ ] CONSTRAINT: **jsdom CANNOT reproduce Safari's grid+abs-pos sizing collapse OR its transform-transition freeze** — D-24 structural assertions (presence/absence of class names, inline style attribute regex) are the closest jsdom can get. They are necessary but not sufficient; visual UAT on real Safari Desktop is required to confirm any Issue-A class fix. Structural mitigation: keep the D-24 structural test pattern alive; add real-Safari UAT as a blocking gate for any future BreathingShape.tsx markup change.

**Do not proceed until all boxes are checked.**
