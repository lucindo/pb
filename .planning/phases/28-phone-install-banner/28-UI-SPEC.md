## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between app icon and banner text (flex gap) |
| sm | 8px | Horizontal padding inside the banner row |
| md | 16px | Horizontal padding on banner sides (left/right edges) |
| lg | 24px | Vertical margin-top from the element above the banner |
| xl | 32px | iOS expanded-steps section top padding |
| 2xl | 48px | Not used in this component |
| 3xl | 64px | Not used in this component |

Exceptions:
- Touch target floor: 44px minimum height and width for dismiss (×) button and
  Android install button and iOS steps-trigger button. This is a WCAG 2.5.5
  hard floor, already established in the codebase (`min-h-[44px] min-w-[44px]`
  seen in StatsFooter.tsx, SettingsStepper.tsx, MuteToggle.tsx).
- App icon display size: 32×32px (img element inside the slim one-line banner).
  Not a spacing token — it is a fixed dimension. Source: banner must be slim;
  32px icon fits within the 44px touch-target row without forcing extra height.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Banner text (one-line) | 14px (text-sm) | 400 (regular) | 1.5 |
| iOS step item | 14px (text-sm) | 400 (regular) | 1.5 |
| Android install button | 14px (text-sm) | 600 (semibold) | 1 (single-line button label) |
| iOS steps-trigger button | 14px (text-sm) | 600 (semibold) | 1 (single-line button label) |

Source: Existing StatsFooter.tsx uses `text-sm leading-6` (14px at 1.5) for
footer text — same density as the banner requires. Existing button labels use
`text-base font-semibold` in dialogs, but those are primary full-width CTAs;
the install button is an inline secondary action inside a slim banner row so
14px semibold fits the available space. No heading role needed — the banner has
no heading, only a single line of descriptive text.

Note: The project uses exactly 2 weights in all existing components: 400
(regular, body/muted text) and 600 (semibold, labels/buttons/headings). This
contract matches that 2-weight rule.

## Color

| Role | Token | Light value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) — banner background | `--color-breathing-bg-soft` | `#e5e9f0` | Banner row background, one shade deeper than page bg so the banner is visually distinct without introducing a new surface color |
| Secondary (30%) — banner border | `--color-breathing-accent` | `#81a1c1` | 1px top border on the banner strip (same border token used on icon-buttons throughout the app) |

Accent reserved for: Android install button label, iOS steps-trigger button

using `bg-soft` for subtle contained surfaces, `accent` for borders and
interactive affordances, and `accent-strong` for primary action text is
established in MuteToggle.tsx and SettingsStepper.tsx.

## Component Layout Contract

### Banner (collapsed / default state)

```
┌─────────────────────────────────────────────────────────────┐  ← 1px border-top,
│ [icon 32×32]  [banner text, text-sm, muted]  [btn]  [×44px] │    bg-soft background
└─────────────────────────────────────────────────────────────┘
```

- Single flex row, `items-center`, `gap-2` (8px) between icon and text
- Banner text fills remaining space (`flex-1`, truncate if needed — but copy
  is short enough to avoid wrapping; see Copywriting Contract)
- Action button (Android: "Install" / iOS: "How to install") is right of text,
  left of × dismiss button
- Dismiss × is the rightmost element; 44×44px touch target
- Total banner height: 56px (padding 6px top/bottom + 44px content row). This
  is the minimum that provides 44px touch targets without visual padding waste.
- Horizontal padding: 16px left and right (px-4)
- Position: normal document flow, bottom of page content, below StatsFooter.
  D-02 source: 28-CONTEXT.md
- Top border: 1px solid `--color-breathing-accent` for visual separation from
  footer above
- No box-shadow — the banner is a subtle secondary element; shadow would
  compete with the app's main card

### iOS Expanded Steps (visible only after tapping "How to install")

```
┌─────────────────────────────────────────────────────────────┐
│ [icon 32×32]  [banner text, text-sm, muted]  [btn]  [×44px] │
├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┤
│  1. Tap the Share button [Share SVG] in Safari's toolbar     │
│  2. Scroll down and tap "Add to Home Screen"                 │
│  3. Tap "Add" to confirm                                     │
└─────────────────────────────────────────────────────────────┘
```

- Steps appear below the banner row (inline expansion, no modal — D-05)
- Steps section has 16px top padding (`pt-4`) and 16px left indent to align
  with banner text (same 16px horizontal padding + 32px icon + 8px gap = 56px
  left offset, OR simply use the same horizontal padding as the banner row for
  simplicity: 16px each side)
- Each step is a single text line, `text-sm` at weight 400, `leading-6`
- iOS Share glyph: inline SVG, 18×18px, `aria-hidden="true"`, displayed inline
  after "Share button" text in step 1 (D-06)
- Step expansion is a React state toggle (`iosExpanded: boolean`); no animation
  required (slim is a hard constraint; the appearance is instant). Under
  `prefers-reduced-motion: reduce` the expand is already instant (no transition
  defined)
- Steps section background: same `--color-breathing-bg-soft` as the banner row
  (no color change on expand)

## Interaction Contract

### States

| State | Condition | Visual |
|-------|-----------|--------|
| Hidden | `appPhase !== 'idle'` OR `isStandalone` OR `dismissed` OR (Android: `deferredPrompt === null`) | Component not rendered |
| Visible — Android | `isPhone && !isStandalone && !dismissed && appPhase === 'idle' && deferredPrompt !== null` | Banner row, "Install" button |
| Visible — iOS (collapsed) | `isPhone && !isStandalone && !dismissed && appPhase === 'idle' && isIOS` | Banner row, "How to install" button |
| Visible — iOS (expanded) | Same as above + `iosExpanded === true` | Banner row + steps section below |
| Dismissed (any platform) | User tapped × | Banner unmounts immediately; `hrv:install-dismissed` written to localStorage |

### Focus behavior

- Dismiss × button: standard `focus-visible:ring-2 focus-visible:ring-breathing-accent` (matches project focus ring convention)
- Android install button: same focus ring
- iOS steps-trigger button: same focus ring
- No focus trap — the banner is a normal document-flow element, not a modal

### Animation

- Show: no animation (D-02 places banner in document flow; it appears as page
  loads if conditions are met)
- iOS expand: no CSS transition — instant height grow (slim constraint means we
  do not animate content expansion that could delay user reading the steps)
- Dismiss: unmounts immediately (no fade-out); follows same pattern as the app's
  conditional rendering elsewhere
- Reduced-motion: no changes needed (no animations defined in this component)

## Copywriting Contract

Source for all copy keys: `src/content/strings.ts` — add `UiStrings.install` block.
No inline copy in the component. See D-09 (28-CONTEXT.md).

| Element | String key | EN copy | Notes |
|---------|-----------|---------|-------|
| Banner body text | `install.bannerText` | "Add to your home screen for offline use" | One line maximum; must not wrap on a 320px viewport. Verified: 43 chars, fits at 14px on 320px minus ~88px for icon+buttons. |
| Android install button | `install.installButton` | "Install" | Verb-only label; mirrors CTA brevity convention in SessionControls ("Start") |
| iOS steps-trigger button | `install.iosStepsButton` | "How to install" | Imperative phrase; expands the steps inline |
| Dismiss × aria-label | `install.dismiss` | "Dismiss install banner" | Screen reader label for the × button; never visible on screen |
| iOS step 1 | `install.iosStep1` | "Tap the Share button in Safari's toolbar" | iOS Share SVG renders inline after "Share button" |
| iOS step 2 | `install.iosStep2` | `'Tap "Add to Home Screen"'` | Exact menu item name; quoted for clarity |
| iOS step 3 | `install.iosStep3` | `'Tap "Add" to confirm'` | Completion step |

PT-BR draft: Required per D-09. Use rough Portuguese translations; add comment
`// DRAFT: Phase 29 will finalize PT-BR install copy` above each PT-BR value in
strings.ts. (Draft values: `install.bannerText` → "Adicione à sua tela inicial para uso offline",
`install.installButton` → "Instalar", `install.iosStepsButton` → "Como instalar",
`install.dismiss` → "Fechar banner de instalação", `install.iosStep1` →
"Toque no botão Compartilhar na barra do Safari",
`install.iosStep2` → "Toque em \"Adicionar à Tela de Início\"",
`install.iosStep3` → "Toque em \"Adicionar\" para confirmar")

Empty state: not applicable — the banner either renders or does not render. There
is no "no data" state to display inside the banner.

Error state: not applicable — if `beforeinstallprompt` never fires (Android),
no banner renders (D-08). No error message is shown to the user.

Destructive actions: none — dismiss is a preference, not a destructive action.
Dismissal is persisted silently with no confirmation dialog (D-04: × tap is
sufficient; D-05 explicitly chose no modal).

## Accessibility Contract

- Banner container: `role="region"` with `aria-label="Install app"` (landmark
  for screen readers navigating by regions)
- App icon: `alt="HRV app icon"` (decorative but meaningful in context)
- Dismiss button: `aria-label={strings.install.dismiss}`
- Android install button: no aria-label needed (label text is the accessible name)
- iOS steps-trigger: no aria-label needed (label text is the accessible name)
- iOS steps section: `aria-live="polite"` on the expand container so screen
  readers announce the step content when it appears
- Step numbers are plain text (`1.`, `2.`, `3.`) — not ARIA list; three items
  is short enough that a numbered text block reads naturally
