<objective>
Build the two detection hooks that gate banner visibility: `useBeforeInstallPrompt` (captures the Android `beforeinstallprompt` event and exposes a replay function) and `useIsStandaloneOrPhone` (detects phone form factor and already-installed state).

Purpose: These hooks isolate non-React browser APIs behind a stable React surface so App.tsx (Plan 03) can compose the `showBanner` boolean from plain booleans/state without touching `window` event plumbing. The Android hook also wires into Plan 01's `saveInstallDismissed` so an install accepted via the browser's own UI permanently hides the banner.

Output: `src/hooks/useBeforeInstallPrompt.ts` and `src/hooks/useIsStandaloneOrPhone.ts`, each with a co-located `.test.ts`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser runtime → app code | The `beforeinstallprompt` / `appinstalled` events and `navigator.standalone` / `matchMedia` results cross from the browser into app state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-04 | Spoofing | useBeforeInstallPrompt — fake beforeinstallprompt via injected script | accept | `beforeinstallprompt` is dispatched only by the browser runtime; a same-origin script could synthesize an `Event`, but `prompt()` on a synthetic object would simply do nothing harmful (no native dialog, no privilege). No secrets exposed. RESEARCH.md Security Domain confirms no mitigation needed. |
| T-28-05 | Tampering | useBeforeInstallPrompt — prompt() called outside user gesture | mitigate | `triggerInstall` performs no async work before `prompt()` and is documented to be called directly from an `onClick`; this preserves the gesture chain (Pitfall 2). The hook never invokes `prompt()` from an effect or timer. |
| T-28-06 | Elevation of Privilege | install prompt | accept | The install action is a browser-native capability with OS-level confirmation; the app cannot install itself without the user accepting the browser's own dialog. No app-side privilege to elevate. |
