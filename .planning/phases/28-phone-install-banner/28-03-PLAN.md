## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-07 | Spoofing | InstallBanner — clickjacking of the install/dismiss controls | mitigate | The banner is rendered in normal document flow between `</section>` and `</main>` (D-02), never as a fixed/absolute overlay, and uses no `pointer-events` or `z-index` manipulation; it cannot overlay or be overlaid to trick clicks. RESEARCH.md Security Domain confirms this disposition. |
| T-28-08 | Tampering | App.tsx showBanner gate | mitigate | `showBanner` is a pure derived boolean from hook outputs and `appPhase`; it has no external mutation surface. The `installDismissed` value is read once via `loadInstallDismissed` whose tamper handling is covered by T-28-01 (Plan 01). |
| T-28-09 | Information Disclosure | InstallBanner UI | accept | The banner displays only static localized copy and the public app icon; no user data, no secrets are rendered. No mitigation needed. |
</threat_model>
