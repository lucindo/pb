<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app code → localStorage | A persisted boolean crosses into browser storage; storage may be unavailable (Safari ITP, private mode, quota) or contain an attacker-tampered value |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | Tampering | installDismissed.ts read path | mitigate | `loadInstallDismissed` strict-equals `'true'`; any other value (including injected JSON or scripts) reads as `false`. No `eval`/`JSON.parse` on the stored value — it is treated as an opaque string. |
| T-28-02 | Denial of Service | installDismissed.ts read/write | accept | localStorage throwing (ITP/quota/private mode) is caught silently per D-16/D-17; worst case the banner re-appears or never persists dismissal — no crash, no data loss of value. Low impact, no PII. |
| T-28-03 | Information Disclosure | hrv:install-dismissed key | accept | The stored value is a single non-sensitive boolean; readable by any same-origin script. No secret, no user data. |
</threat_model>
