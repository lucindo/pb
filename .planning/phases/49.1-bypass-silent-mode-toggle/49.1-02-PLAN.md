<objective>
Gate the Phase 49 silent-loop `<audio>` element on a new
`opts.bypassSilentMode` engine option (D-07), and thread the flag through
`useAudioCues.start` (3rd arg) and `useBreathingSessionController` so the
eventual wire-up from `useAppViewModel` (Plan 03) reads
`featureFlags.bypassSilentMode` end-to-end.

The gate is binary: `false` skips construction entirely (no `new Audio(...)`,
no `.play()` call, no teardown branch needed because the existing
`if (silentLoopElement !== null)` null-guards in close() and the resume-reject
catch already short-circuit). `true` or `undefined` is byte-identical to
Phase 49 v3 ‚Äî preserving the shipped iOS-bypass posture for any caller that
hasn't been updated yet.

Per D-09, the toggle is read at engine-construction time only (Start /
reconstructEngine), NOT live during a session ‚Äî rebuilding the engine on a
preference toggle would interrupt the practice. This matches the existing
`timbreRef` capture-and-replay pattern.

Purpose: the audio half of Phase 49.1. Plan 01 publishes the flag; this
plan teaches the audio engine to honor it. Plan 03 closes the end-to-end
loop by wiring `useAppViewModel` to pass the flag down.

Output: gated `createAudioEngine`, 3 new SpyAudio tests covering ADV-04,
`useAudioCues.start` accepts a 3rd arg, and `useBreathingSessionController`
accepts a new optional prop + forwards it to `audioStart` line 185 with the
existing capture-and-replay semantics.
</objective>
