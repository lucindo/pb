| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-value inline spacing |
| sm | 8px | Intra-component element gaps |
| md-tight | 12px (Tailwind `*-3`) | Stepper inner row gap, between-button gap, mid-gradation vertical rhythm |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section gaps, modal inner padding |
| xl | 32px | Layout vertical rhythm |
| 2xl | 48px | Orb top/bottom margins |
| 3xl | 64px | — (not used in this phase) |

**Note on 12px (md-tight):** Tailwind's natural `*-3` step (12px) sits between the canonical 8/16 stops and is used liberally in shipped code (`gap-3`, `py-3`, `mt-3`). Declared explicitly here so the spacing audit treats it as in-scale rather than as an arbitrary deviation.

### Orb Color Contract (D-02)

| Phase | Gradient | Text Color | Meaning |
|-------|----------|------------|---------|
| In (inhale) | `linear-gradient(135deg, #ccfbf1, #ecfdf5)` — warm teal-to-green | `#134e4a` (deep teal) | Expanding, warm |
| Out (exhale) | `linear-gradient(135deg, #dbeafe, #eef2ff)` — cool blue-to-indigo | `#1e3a8a` (deep blue) | Contracting, cool |

**Color transitions:** Implemented via two stacked gradient `<div>` layers (In layer always visible, Out layer `opacity` 0→1 on phase change). CSS `opacity` transition at 400ms `ease-in-out`. This is the only supported approach — `background-image` is not transitionable in CSS (RESEARCH.md).

**Non-color cues required (GUID-04, D-02):** Phase is ALWAYS communicated by both color shift AND the text label (`In`/`Out`) inside the orb. Color alone is never the sole indicator.

## Interaction Contracts

### Orb Animation (GUID-01, D-01–D-04, D-18)

| Interaction | Normal motion | Reduced-motion |
|-------------|---------------|----------------|
| Inhale phase progress | Orb scale 0.58→1.0 via `transform: scale()` over full phase duration; transition `duration-200 ease-out` per-tick; `background-image` stays In-gradient | Orb fixed at `scale(0.79)` — no scale change |
| Exhale phase progress | Orb scale 1.0→0.58; same transition | Orb fixed at `scale(0.79)` |
| Phase change (In→Out) | Orb scale reversal + Out gradient layer opacity 0→1 at 400ms ease-in-out; label swaps | Out gradient layer opacity 0→1 at 400ms ease-in-out; label swaps (D-07) |
| Phase change (Out→In) | Out gradient layer opacity 1→0 at 400ms ease-in-out; scale reversal; label swaps | Out gradient layer opacity 1→0 at 400ms ease-in-out; label swaps |
| Idle state (no session) | Orb not rendered (frame is null) | Orb not rendered |

**Scale bounds:** 0.58 (min-exhale) to 1.0 (max-inhale). Outer static ring at 1.0. Inner static ring at 0.58 of orb size (positioned absolutely centered, width = height = 58% of container).

**Easing:** Scale transitions use `ease-out` (decelerate into breath boundary). Gradient crossfade uses `ease-in-out` (symmetric fade).

**Duration:** Scale per-tick transition: 200ms (matches existing Phase 1 baseline). Gradient crossfade: 400ms (within D-07 300–500ms band).

**motion-reduce global scope (D-09):** All button/stepper/focus-ring decorative transitions disabled via `motion-reduce:transition-none motion-reduce:duration-0` where applicable.

### Confirmation Modal (D-10–D-14)

| Trigger | Behavior |
|---------|----------|
| User taps "End session" with timed session running | Modal opens via `dialog.showModal()`; backdrop dims; focus moves to "Keep going" button |
| User presses Esc | `cancel` event fires; modal closes; focus returns to "End session" button; session continues |
| User clicks "Keep going" | `dialog.close()`; focus returns to "End session" button; session continues |
| User clicks "End" | `dialog.close()`; `session.end()` is called; session ends |
| User clicks backdrop (outside dialog panel) | Same as "Keep going" |
| User taps "End session" with open-ended session running | No modal; `session.end()` called directly (D-14) |
| Session clock during open modal | Continues running; orb and readout update live behind backdrop (D-13) |

**Modal entry:** Fade in at 200ms `ease-out` (`@starting-style` or opacity transition from 0). **Normal motion only** — in reduced-motion, dialog appears immediately without transition.

**Modal exit:** Fade out at 150ms `ease-in`. Reduced-motion: immediate.

**Modal max-width:** `max-w-sm` (384px) on `sm` and up. Full-bleed minus 32px horizontal margin on mobile.

### Focus Ring Contract (GUID-04, D-21)

Applied to: all buttons (stepper +/-, Start session, End session, End, Keep going), fieldsets with keyboard-accessible children.

```
focus-visible:outline-none
focus-visible:ring-2
focus-visible:ring-[--color-breathing-accent]   (= #0f766e)
focus-visible:ring-offset-2
```

Tailwind v4 auto-generates `ring-breathing-accent` from `--color-breathing-accent` in `@theme`. Use `focus-visible:ring-breathing-accent`.

Ring does NOT appear on mouse click (`:focus-visible` pseudo-class, not `:focus`).

**Remove from Phase 1:** `focus:outline-none focus:ring-4 focus:ring-teal-200` on stepper buttons. Replace with the `focus-visible:` pattern above.

### Running Session Layout (D-15–D-16, D-19, MOBL-01)

| State | Visible elements (top → bottom) |
|-------|----------------------------------|
| Idle | Header copy → Orb (null, hidden) → Settings (BPM + Ratio + Duration) → Start session |
| Running (timed) | Header copy → Orb (animated) → Session readout (clock) → Duration display only → End session |
| Running (open-ended) | Header copy → Orb (animated) → Session readout (clock) → "Open-ended" label only → End session |
| Complete | Header copy → Session readout ("Session complete") → Settings (all visible) → Start session |

BPM stepper and Ratio stepper are conditionally rendered absent when `isRunning === true` (not hidden with CSS visibility — removed from DOM so screen readers do not encounter locked controls).

## Reduced-Motion Contract (GUID-03, D-05–D-09)

Detection: `window.matchMedia('(prefers-reduced-motion: reduce)')` via `usePrefersReducedMotion` hook. Reactive — updates if the OS preference changes during a session.

| Element | Normal motion | Reduced-motion |
|---------|---------------|----------------|
| Orb scale | Continuously animates with `phaseProgress` | Fixed at `scale(0.79)` (mid-point of 0.58–1.0) |
| Orb gradient crossfade | 400ms `ease-in-out` opacity crossfade | 400ms `ease-in-out` opacity crossfade (preserved — this is the phase indicator in reduced-motion) |
| Phase label inside orb | Updates in sync with gradient | Updates in sync with gradient |
| Static reference rings | Always visible | Always visible |
| Button hover/active transitions | CSS `transition` decorative | Disabled via `motion-reduce:transition-none` |
| Stepper transitions | CSS `transition` decorative | Disabled |
| Focus ring transitions | CSS `transition` decorative | Disabled |
| Modal entry/exit | 200ms/150ms fade | Immediate, no animation |

**CSS implementation:** Tailwind `motion-reduce:` variants are used for globally disabling decorative transitions. The `usePrefersReducedMotion` hook is used in `BreathingShape` to branch the scale logic (JS-level, since CSS cannot branch a `style` prop value).

## Animation Implementation Details

### CSS Classes (theme.css additions)

```css
/* Phase gradient layers */
.orb-layer--in {
  background: linear-gradient(135deg, var(--color-orb-in-from), var(--color-orb-in-to));
}
.orb-layer--out {
  background: linear-gradient(135deg, var(--color-orb-out-from), var(--color-orb-out-to));
  opacity: 0;
  transition: opacity 400ms ease-in-out;
}
[data-phase='out'] .orb-layer--out {
  opacity: 1;
}

@media (prefers-reduced-motion: reduce) {
  /* Scale suppressed via JS (usePrefersReducedMotion → orbScale = mid).
     Gradient crossfade is intentionally PRESERVED — it is the phase
     indicator for reduced-motion users (D-07). */
  .breathing-shape,
  .orb {
    transform: none !important;
    transition: none !important;
  }
  /* Only the opacity crossfade is kept — handled at .orb-layer--out level */
}
```

**Note on scale animation:** Scale is applied via React `style` prop (`transform: scale(${orbScale})`), not a CSS class. Tailwind `motion-reduce:transition-none` suppresses the `transition-transform` utility. In reduced-motion mode, `orbScale` is fixed at the mid-point — no CSS override needed for the value, only for the transition duration.

### Tailwind Utilities Composition (Orb)

```
className="
  relative mx-auto grid place-items-center
  rounded-full
  transition-transform duration-200 ease-out
  motion-reduce:transition-none
"
style={{ width: 'clamp(180px, 35vw, 360px)', height: 'clamp(180px, 35vw, 360px)', transform: `scale(${orbScale})` }}
```

### Tailwind Utilities Composition (Focus Ring — all interactive controls)

```
focus-visible:outline-none
focus-visible:ring-2
focus-visible:ring-breathing-accent
focus-visible:ring-offset-2
motion-reduce:transition-none
```
