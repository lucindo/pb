## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| worktree-merge-branch-drift | Wave-2 merge loop iterated two worktrees and HEAD drifted off `main` onto the second worktree's branch; wave-2 commits landed on stray branch, not main. | blocking | Per-iteration `git rev-parse --abbrev-ref HEAD` assertion before AND after each `git merge`. Abort and surface to user if drifted. |
| executor-sandbox-planning-git-add-denial | Subagent's sandbox denied `git add` for paths under `.planning/` while permitting same operation under `src/`. SUMMARY.md uncommitted, would have been lost on `git worktree remove --force`. | blocking | Spawn prompt MUST instruct fallback variants AND require explicit `SUMMARY UNCOMMITTED` marker on failure. Orchestrator must always run the worktree SUMMARY rescue snippet (find + cp) before `git worktree remove`. |
| reduced-motion-css-override-mutes-inline-transform | `transform: none !important` in a reduced-motion CSS branch silently overrides any inline `transform:` value, producing a different rendered scale than tests assert (because tests check inline string, not computed style). | advisory | Code-review checked computed-vs-inline mismatch this round (CR-01). When asserting style under reduced motion, prefer `getComputedStyle(...)` or visual screenshot tests over `el.style.transform`. |
