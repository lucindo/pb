<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dialog → external browser tab | External Forrest links navigate the user away from the practice tab. |
| user input (Esc / Enter / backdrop) → dialog state | Native dialog default-focus behavior can route Enter to whichever element has focus. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-07 | Tampering | External-link tabnabbing | mitigate | Every `<a>` rendered inside the link block sets `target="_blank"` AND `rel="noopener noreferrer"` (CONTEXT.md D-07). Tests assert both attributes on every link. |
| T-06-08 | Elevation of Privilege | Accidental Enter dispatch on link | mitigate | Default focus on dialog open lands on the Close button (CONTEXT.md D-05). Test asserts `screen.getByRole('button', { name: 'Close' }).toHaveFocus()` immediately after `dialog.showModal()`. |
| T-06-09 | Repudiation | Disclaimer copy drift | mitigate | The two disclaimer strings are inline JSX (per CONTEXT.md §Established Patterns "Inline locked copy"); tests assert both literal strings render in the DOM. |
| T-06-10 | Denial of Service | Third-party script execution | accept | No iframe, no embed, no third-party script ships with the dialog (D-13 plain text labels only). |

</threat_model>
