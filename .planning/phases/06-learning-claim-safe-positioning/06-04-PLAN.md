<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anchor click → modal state | A click during session view must NOT transition the modal to open. |
| iOS safe-area inset → fixed-position element | Anchor at the top edge of the viewport must respect iOS notch/dynamic island insets. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-11 | Tampering | Modal-open during session | mitigate (defense in depth) | Two-layer gate: (a) anchor renders with `onClick={disabled ? undefined : onClick}` so the DOM event never fires during session view; (b) App's `onLearnClick` early-returns when `inSessionView` is true. Tests assert both layers. |
| T-06-12 | Information Disclosure | Anchor unmount mid-session | accept (turned into invariant) | D-01 requires the anchor to be persistent across all states (never unmounted). Test asserts the same anchor DOM node identity persists across an `inSessionView` transition. |
| T-06-13 | Denial of Service | iOS safe-area overlap | mitigate | `top-[max(1rem,env(safe-area-inset-top))] right-[max(1rem,env(safe-area-inset-right))]` per UI-SPEC §Layout Contract. Verified during execute-phase visual sweep (deferred to standard regression — D-20 says no blocking UAT for Phase 6). |
