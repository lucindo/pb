<objective>
Resolve LEAD-01: prevent an accidental double-start during the 3-second lead-in
countdown by swapping the primary button label to `Cancel` / `Cancelar` while
`appPhase === 'lead-in'`. The button's click already routes to the existing
cancel branch (`onStartClick` at `App.tsx:312`); this plan only makes that
already-working cancel intent legible by relabeling the button.

Purpose: A button that still reads "Start session" during the countdown invites
a second click that the user reads as "start again." Relabeling to "Cancel"
removes the double-start affordance while keeping the existing cancel capability
discoverable (per D-01).

Output: A `cancel` string in both locales, an `inLeadIn?: boolean` prop on
`SessionControls` with three-way label resolution, the `App.tsx` wiring that
feeds it, the corrected Phase 3 copy-lock comment, the rewritten Test 11, and a
positive regression test for the lead-in label in both locales.
</objective>
