# Phase 48 — UI Design Contract

> Visual and interaction contract for the new Appearance page + the App Settings header right-chevron + the EN/PT-BR copy for every new string. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Composition-only phase against a locked design system (spike-010 Mono Zen, closed at v2.0).** This spec restates the locks the page must honor, declares the exact layout / a11y / copy contract for the new surfaces, and lists the dimensions the checker will verify. It does **not** introduce new tokens, new primitives, new typography, or new color values. All primitives (`SegmentedControl`, `SettingsToggleRow`, `PageShell`, `TopAppBar`, `IconButton`, `SettingsSectionHeader`, `ChevronRightIcon`, `ChevronBackIcon`) and the inline `SectionCard` chrome are reused verbatim from existing consumers (`AppSettingsPage`, `LanguagePicker`, `SettingsPanelBody`).

## Layout Contract

### A. App Settings page — additive change (right-chevron in trailing slot)

**Affected file:** `src/app/pages/AppSettingsPage.tsx`

**Change:** add a `trailing` slot to the existing `TopAppBar`. The trailing slot holds an `IconButton` with the `ChevronRightIcon`, `aria-label="Appearance settings"`, `onClick={onAppearanceOpen}`, and a `buttonRef={chevronButtonRef}` for focus restoration.

```
TopAppBar:  [ < (back) ]    Settings    [ > (appearance) ]
            ^leading slot               ^trailing slot
```

- **Visual symmetry:** both slots are 36×36 (`size-9`) — verified by `IconButton` defaulting to `md` (40px) but `TopAppBar` reserving `size-9` placeholders for empty slots. **Use `IconButton` default size (`md` = 40px)** to match the existing back-chevron's visual weight — `AppSettingsPage` currently passes no `size` prop, defaulting to `md`. Phase 48 right-chevron mirrors this.
- **Affordance gating:** `onAppearanceOpen` is gated by `controlsDisabled` inside `useAppNavigation` (same gate as `onSettingsOpen`/`onLearnOpen`). The chevron remains visually present and tappable on App Settings — App Settings itself is only reachable when `controlsDisabled === false`, so the gate is conceptually a no-op here but kept for symmetry.
- **Focus management on mount:** the existing `useEffect` on `AppSettingsPage` that focuses `backButtonRef` becomes conditional on `returningFromAppearance`:
  - `returningFromAppearance === true` → focus `chevronButtonRef` (D-13 contract).
  - `returningFromAppearance === false` (default, fresh entry from `Settings` icon) → focus `backButtonRef` (existing behavior).

### B. Appearance page — new full-page surface

**New file:** `src/app/pages/AppearancePage.tsx` (+ `.test.tsx`)

**Chrome (verbatim paste-and-rename of `AppSettingsPage`):**

```
PageShell (width="page", 600px cap on desktop, full-width mobile)
  TopAppBar
    leading:  IconButton(ChevronBackIcon, aria-label="Back to Settings",
                          onClick={onBack}, buttonRef={backButtonRef})
    title:    "Appearance"
    trailing: (empty — 36×36 placeholder for title centering)

  <div className="w-full text-left">
    SettingsSectionHeader (as="h2"): "Orb Style"
    SectionCard (padding="16px"):
      <div className="grid gap-4">
        OrbPicker
          (sectionLabel="Orb", sectionLabelHidden={false},
           disabled={false}, strings={appearance.orb.options})
        RingCuePicker
          (sectionLabel="Ring cue", sectionLabelHidden={false},
           disabled={false}, strings={appearance.ringCue.options})
      </div>

    SettingsSectionHeader (as="h2"): "Visual"
    SectionCard (padding="16px"):
      SettingsToggleRow
        label="Breathing effect"
        ariaLabel="Breathing effect"
        checked={orbIdle === 'ambient'}
        onChange={(next) => setOrbIdle(next ? 'ambient' : 'still')}
      SettingsToggleRow
        label="Switcher icons"
        ariaLabel="Switcher icons"
        checked={switcherIcon}
        onChange={setSwitcherIcon}
  </div>
```

**Section order rationale:** `Orb Style` first (visual identity of the practice surface — most impactful change), `Visual` second (subtle motion + chrome toggles). Matches the operator's mental model where the orb is the central practice element and the toggles are secondary refinements.

**Picker label visibility:** `sectionLabelHidden={false}` for both pickers because the Orb Style card holds **two** pickers; each needs its own visible label (`Orb` and `Ring cue`) to disambiguate. This differs from the App Settings precedent where the Language card has one picker and uses `sectionLabelHidden={true}` (the section header `Language` is sufficient). **The Orb Style card is the first multi-picker card in this app** — the precedent is the App Settings Audio card (`CuePicker` + `TimbrePicker`), which also renders both labels visibly.

**Toggle layout:** two `SettingsToggleRow` instances stacked inside one card. Each row composes `SettingsRow`'s `border-t border-[var(--color-border-soft)] py-3` divider, producing a hairline between the two toggles — same chrome as the legacy `SettingsStepper` rows. **No `noBorder` prop** on either row.

## Interaction & Focus Contract

### Navigation flows

1. **Enter Appearance from App Settings:** user presses the right-chevron `IconButton` in `TopAppBar` trailing. `onAppearanceOpen()` clears the `returningFromAppearance` sentinel and transitions `AppScreen` to `'appearance'`. `ScreenRouter` renders `<AppearancePage onBack={vm.dialogs.onBackToAppSettings} />`. **On mount, the page focuses `backButtonRef`** (the back-chevron) via `useEffect(() => backButtonRef.current?.focus({ preventScroll: true }), [])`.

2. **Exit Appearance back to App Settings:** user presses the back-chevron `IconButton`. `onBack` invokes `vm.dialogs.onBackToAppSettings()` which sets `AppScreen` to `'appSettings'` and `returningFromAppearance: true`. `ScreenRouter` re-renders App Settings; App Settings' mount `useEffect` reads `returningFromAppearance` and **focuses `chevronButtonRef`** (the right-chevron source). Sentinel is cleared by the next non-back-to-AppSettings navigation (or after consumption per D-13).

3. **Session-start interrupt:** if a practice session starts while the user is on Appearance, the existing `closeOnSessionView` effect in `useAppNavigation` forces `AppScreen` back to `'practice'` and clears the sentinel — same behavior as Learn and App Settings. **No special handling required for Appearance.**

### Focus visibility

Every interactive element in this phase uses the existing `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-breathing-accent focus-visible:ring-offset-2` recipe (encoded by the reused primitives — `IconButton`, `SegmentedControl` buttons, `Toggle`). **No new focus styling.**

### Reduced-motion contract

- `SegmentedControl` active-pill transitions use `transition motion-reduce:transition-none` — preserved.
- `Toggle` knob slide uses `transition-[left] duration-[180ms] motion-reduce:transition-none` — preserved.
- `IconButton` hover/active transitions are also `motion-reduce:transition-none` — preserved.

**No new motion in this phase; reduced-motion is fully honored by the reused primitives.**

### Live-update propagation contract (APPEAR-05)

When the user touches any picker or toggle on the Appearance page:

1. The choice-hook setter (e.g. `setBreathingShape(next)`) is called — this is the `onChange` prop of `SegmentedControl` / `SettingsToggleRow`.
2. The setter (verified at `src/hooks/useBreathingShapeChoice.ts` and the three siblings) (a) writes through `savePrefs`, (b) updates local React state, (c) dispatches `window.dispatchEvent(new CustomEvent('hrv:prefs-changed', { detail: { key, value } }))`.
3. The `useFeatureFlags` listener (already wired in Phase 47 at `src/hooks/useFeatureFlags.ts:73-92`) filters on the four-key set and re-resolves `FeatureFlags`.
4. `PracticeScreen` re-renders with the new `vm.featureFlags`; the practice orb / ring cue / idle mode / switcher chrome update on the next render.

**Phase 48 ships ZERO new event-dispatch sites.** All propagation is owned by Phase 47. The picker/toggle `onChange` MUST call the choice-hook setter directly — no ad-hoc `dispatchEvent` calls (per RESEARCH.md Pitfall 3).
