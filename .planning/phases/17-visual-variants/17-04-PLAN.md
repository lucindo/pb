<objective>
Create the two hooks that mirror Phase 16's theme orchestration pattern (`useTheme` / `useThemeChoice`) for the variant dimension:

- **`useVisualVariant`** — App-side orchestrator. Seeds state from `loadPrefs().variant`, subscribes to cross-tab `'storage'` events filtered on `STATE_KEY`, subscribes to same-tab `'hrv:prefs-changed'` events filtered on `detail.key === 'variant'` (with the `detail.key === undefined` broadcast-all branch preserved per D-22). Per D-16 — does NOT write any global attribute (no `<html data-variant>`); does NOT subscribe to matchMedia (variant is not OS-driven).

- **`useVariantChoice`** — Picker-side setter. Verbatim mirror of `useThemeChoice` with `theme` → `variant` substitutions: 4-step write (fresh `loadPrefs()` / merged `savePrefs()` / local `setVariantState()` / dispatch `'hrv:prefs-changed'` CustomEvent with `detail: { key: 'variant', value: next }`).

Both hooks are independent of the shape components (Plan 03) — Plan 04 can land in parallel with Plan 03 as wave 3. Plan 05 (BreathingShape dispatcher + VariantPicker body) consumes both hooks.

Output: 2 hook files + 2 test files, all token + signature compatible with Phase 16 conventions; full test suite green at commit boundary.
</objective>
