| Boundary | Description |
|----------|-------------|
| App → YouTube / external sites (Learn links) | Every Learn link `<a>` navigates the user to a third-party origin in a new tab |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-03 | Information Disclosure | NK video `<a>` elements (reverse-tabnabbing / referrer leak) | mitigate | Every `<a>` in `LearnDialog.tsx`, including the two new NK video links, carries `target="_blank" rel="noopener noreferrer"` — `noopener` prevents the opened page accessing `window.opener`, `noreferrer` suppresses referrer. Enforced by the parameterized every-`<a>` security-attribute test run for `activePractice='naviKriya'` in Task 2. |
| T-32-04 | Tampering | `activePractice` prop value | accept | `activePractice` is a `PracticeId` union (`'resonant' | 'naviKriya'`) sourced from `App.tsx` typed state — not user free-text. TypeScript constrains it; `practices[activePractice]` access is exhaustive over the union. |

No high-severity threats. This plan adds prop-driven conditional rendering with no network calls, no user free-text input, no auth, no new persistence.
</threat_model>
