## Practice-Aware Layout Contract

### Section Order (D-01)

When a practice is active, LearnDialog renders sections in this fixed order:

```
1. [PRACTICE-SCOPED] Practice description  ← active practice identity (D-08)
2. [PRACTICE-SCOPED] Practice videos       ← active practice only
3. [SHARED] Forrest Knutson Resources      ← always present, both practices
4. [SHARED] Native apps block              ← resonant ONLY (D-02)
5. [SHARED] Affiliation micro-line         ← always present, both practices
6. Close button
```

The dialog title stays `strings.learn.title` ("About this practice") — generic, unchanged (D-04).

### Practice-Scoped: Practice Description (D-03, D-08)

The practice description renders as **two parallel sections**, matching resonant's `hrv` + `timing` explainer shape:

**Resonant Breathing (existing):**
- Section 1: `explainer.hrv` — "What is HRV / resonance breathing"
- Section 2: `explainer.timing` — "How this app times your breath"

**Navi Kriya (new — D-03):**
- Section 1: Navi Kriya identity — heading carries "What is Navi Kriya" (sole practice signal per D-08)
- Section 2: App pacing — heading carries "How this app paces it" (mirroring resonant's timing section shape)

Both sections render as: `<h3>` heading in `accent-strong` + `<p>` body in `muted` at `text-base leading-6`.

The locked `inspiredByForrest` phrase is NOT in the Navi Kriya description sections — it lives only in the shared "Who is Forrest" section via `lockedCopy.inspiredByForrest` (unchanged pattern from resonant).

### Practice-Scoped: Practice Videos

**Resonant Breathing (existing):**
- Sub-section heading: `strings.learn.videosHeading` ("Selected HRV Breathing Videos")
- Links: `heroVideo` + `keyVideos[]` (4 total currently)

**Navi Kriya (new — D-06):**
- Sub-section heading: new string key (e.g. `strings.learn.naviKriyaVideosHeading`) — see Copywriting Contract
- Links: exactly two, in this order:
  1. "The Guardian In Meditation" → `https://www.youtube.com/watch?v=M3t7gY_yak8`
  2. "Navi Kriya Walkthrough" → `https://www.youtube.com/watch?v=A4BGQCIp9fI`
- No padding to match resonant's count. Fewer-is-fine (D-06).
- Video titles stay in English in both EN and PT-BR catalogs (D-12, consistent with existing resonant video title pattern).

All video links render identically to existing links: `inline-flex min-h-[44px] items-center text-base font-medium` in `accent` color with hover to `accent-strong`, focus ring on `breathing-accent`.

### Shared: Forrest Knutson Resources

Unchanged. Present for both practices. Render order:
`youtubeChannel` → `website` → `book` → `patreon`

Heading: `strings.learn.resourcesHeading` ("Forrest Knutson Resources")

The "Who is Forrest" explainer section (`explainer.forrest`) renders directly above this block,
closing with `lockedCopy.inspiredByForrest` in italic — unchanged pattern.

### Shared: Native Apps Block (Resonant-Only — D-02)

The native-apps block (`strings.learn.nativeAppsHeading`, "Resonant Breathing app") is **conditionally rendered**:
- Resonant active: render the block (App Store + Google Play links)
- Navi Kriya active: **omit the block entirely** — no heading, no links, no placeholder

No visual gap or placeholder is left where the block was; the affiliation micro-line follows directly after the Forrest Resources block.

### Shared: Affiliation Micro-Line

Present for both practices. Unchanged: `lockedCopy.affiliationLine`, `text-xs text-center muted`.

## Interaction Contract

### Auto-Tracking (D-07)

LearnDialog receives practice-scoped content from `App.tsx` via `activePractice`. When the
user switches practice and opens Learn, the dialog reflects the newly active practice's content.
There is no in-dialog practice toggle — one Learn anchor, no practice switcher inside the dialog.

### Open / Close Behavior

Unchanged from existing implementation:
- Native `<dialog>` with `showModal()` / `close()` imperatively controlled.
- `dialog.scrollTop = 0` reset on open (iOS Safari scroll fix preserved).
- Default focus: Close button (`closeButtonRef.current?.focus()`).
- Backdrop click closes the dialog.
- Esc key fires `cancel` → `onClose()` (preventDefault on cancel event).
- Entry/exit: `modal-fade` class — 200ms opacity fade (`ease-out`); suppressed under `prefers-reduced-motion`.

### Scrollable Content

Dialog uses `overflow-y-auto` with `max-h-[calc(100dvh-2rem)]`. Navi Kriya dialog has fewer
sections than resonant (no native-apps block, only 2 video links) — it will be shorter. No
layout change required; existing overflow handling is sufficient.

### Link Behavior

All external links: `target="_blank" rel="noopener noreferrer"`. Unchanged from existing pattern.

## Content Architecture Contract

### `learnContent.ts` Extension Shape

The per-practice partition adds a `practices` map keyed by `PracticeId` over the shared base.
Exact data shape is a planner decision (flat extension vs. nested `practices` map — see CONTEXT.md
"Claude's Discretion"). The UI contract is practice-keyed access at render time.

Each practice entry provides:
- `description: { section1: ExplainerSection; section2: ExplainerSection }` — the two-section explainer
- `videos: LearnLink[]` — practice-specific video links (resonant: heroVideo + keyVideos; Navi Kriya: exactly 2)

Shared base (unchanged in `learnContent.ts`):
- `explainer.forrest` — "Who is Forrest Knutson" (shared)
- `links.youtubeChannel`, `links.website`, `links.book`, `links.patreon` — Forrest Resources (shared)
- `links.appStoreIos`, `links.googlePlayAndroid` — resonant-only (D-02; rendered conditionally)

### `LearnDialog.tsx` Props Change

The dialog receives practice-scoped content from `App.tsx`. The `activePractice` value drives
which description and video block is passed. Exact prop shape is a planner decision. The existing
`learnContent` prop may be replaced with a practice-aware variant, or supplemented with
`activePractice: PracticeId` to drive conditional rendering.
