<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user click → App state | Pill clicks and reset-confirm clicks cross into app state; timing matters (no switch mid-session). |
| localStorage → App state | App reads migrated per-practice data on mount and on cross-tab `storage` events; all values untrusted until coerced (handled in plan 30-03). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-09 | Tampering | `onSwitchPractice` mid-session | mitigate | `if (inSessionView) return` guard inside the handler is defense-in-depth behind PracticeToggle's `disabled` attribute, so no practice switch can occur while a session runs (PRACTICE-03). |
| T-30-10 | Tampering / data integrity | cross-tab `storage` event listener | mitigate | Listener reads `loadPractices()` (coerced) and refreshes both stats slices, so a cross-tab write cannot zero the displayed stats via the orphaned flat `env.stats` (Pitfall 6). |
| T-30-11 | Repudiation | `confirmReset` wiping the wrong subtree | mitigate | `confirmReset` calls `resetPracticeStats(activePractice)` and updates only the active slice, so a reset cannot silently destroy the other practice's history (D-08). |
| T-30-12 | Denial of service | `npm run build` regression | accept | Static client-only PWA build; a build failure is caught by the Task 2 verify gate before the phase closes. No runtime DoS surface. |
</threat_model>
