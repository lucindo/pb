<objective>
Migrate `useAmbientScale` off direct `performance.now()` reads onto `SessionClock.now()` for the initial `start` capture. The hook accepts a new `wallClock: SessionClock` parameter. Constants INHALE_MS / EXHALE_MS are renamed to INHALE_SEC (4.4) / EXHALE_SEC (6.6) per D-02; rAF math is in seconds. The HOOKS-04 / Plan 06 D-10 cancel-guard idiom (`let cancelled = false` + top-of-tick check + cancel-on-cleanup) is preserved verbatim.

**Revision 1 Warning #8 — preserve rAF DOMHighResTimeStamp for byte-identicality:** The original Plan 50-05 dropped the rAF `tick(now)` timestamp parameter entirely and read `wallClock.now()` at the top of every tick body. That created sub-frame divergence: the rAF timestamp corresponds to the EXACT frame's drawing moment, while `wallClock.now()` (= `performance.now() / 1000`) measured at the top of the callback body is microseconds later than the frame timestamp.

The fix: **preserve the rAF DOMHighResTimeStamp.** Keep the `tick(now: number) => void` signature; convert at the boundary via `now / 1000` to get the seconds-shaped per-tick time source. The INITIAL `start` capture (before the first rAF tick) uses `wallClock.now()` — the only `wallClock.now()` call in the hook. After that, every per-tick read uses the rAF timestamp converted to seconds. Byte-identicality is preserved: the oscillation curve is computed against the same time source as pre-refactor (the rAF timestamp).

**The drift-guard (Plan 50-07) does NOT flag this pattern.** The banned patterns are:
- `\bperformance\.now\(` — a direct CALL to performance.now (the rAF callback parameter `now` is not a call; the rAF runtime supplies the timestamp).
- `\bnew\s+AudioContext\b` — N/A.
- `\baudioCtx\.currentTime\b` — N/A.

The rAF parameter `now` is none of these. The hook reads `now / 1000` (a parameter conversion, not a `performance.now()` call), and the drift-guard regex does not match this. Plan 50-07's drift-guard fs-scans for the three banned patterns and passes.

Per D-09: at the end of this plan, `src/hooks/useAmbientScale.ts` contains zero `performance.now()` CALLS. The wall clock factory (Plan 50-01's `createWallSessionClock`) returns `performance.now() / 1000`, making the substitution byte-identical at runtime for the initial `start` capture.

`OrbShape.tsx` is the sole caller — it constructs a memoized `createWallSessionClock()` instance and threads it through.

Purpose: ABSTR-03 partial satisfaction (`useAmbientScale` is the final caller — caller 5 of 5). ABSTR-04 maintained — test parity preserved.

Output: useAmbientScale's initial `start` reads wallClock.now(); per-tick reads use rAF timestamp / 1000 (revision 1 Warning #8 — byte-identicality preserved); OrbShape threads the wall clock; tests assert seconds-shaped behavior; full suite green.
