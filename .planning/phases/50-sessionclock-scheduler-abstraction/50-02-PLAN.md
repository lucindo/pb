<objective>
Migrate `useSessionEngine` and its domain helpers (`startSession`, `startStretchSession`, `extendTimedSession`, `completeIfNeeded`) off `performance.now()` and onto `SessionClock.now()`, converting all time math from milliseconds to seconds per D-02. This eliminates one of the three clocks (`performance.now()`) from the session-engine surface; Phase 51 will rebase the source onto `audioCtx.currentTime` via the same `SessionClock` seam (D-03/D-10) without further caller changes.

**Revision 1 (Blocker #4 — FP precision risk resolved via path (a)):** The original plan introduced `/1000` conversions inside `getSessionFrame` / `getStretchFrame` (e.g., `plan.inhaleMs / 1000`) for odd-numbered ms values that are not exactly representable in IEEE-754 (e.g., `4350 / 1000 = 4.35` floats with epsilon error). This compounds across rAF ticks and could diverge from the ms-shaped baseline.

The fix: **push the rename to the BreathingPlan source.** `BreathingPlan.inhaleMs` / `exhaleMs` / `cycleMs` / `totalMs` become `inhaleSec` / `exhaleSec` / `cycleSec` / `totalSec` at the type level via LSP rename. `createBreathingPlan` divides by 1000 ONCE at construction time; downstream consumers see seconds end-to-end. There is no runtime `/1000` conversion to introduce epsilon drift. Byte-identicality is **structural** — no FP-equivalence acceptance test needed because no FP conversion happens at runtime.

**Revision 2 Blocker #2 — completes the source-level rename through StretchSegment AND `computeStretchTotalMs`.** Iteration-1 path (a) covered BreathingPlan but stopped short: `StretchSegment.startMs` / `endMs` / `cycleMs` / `inhaleMs` / `exhaleMs` and the exported `computeStretchTotalMs(settings): number | null` were left ms-shaped, leaving a MIXED-UNIT interface across the stretch pipeline (e.g., post-iteration-1, a partial rename would have left `StretchSegment` with `startMs` ms coexisting with new `cycleSec` sec fields — a structural footgun). The plan-checker's iteration-2 verification correctly identified this as a Blocker.

Revision 2 commits to **completing the source-level rename uniformly** through the entire stretch pipeline:
- `StretchSegment.startMs → startSec`, `endMs → endSec`, `cycleMs → cycleSec`, `inhaleMs → inhaleSec`, `exhaleMs → exhaleSec`
- Local `cursorMs → cursorSec`, `durationMs → durationSec` (inside `makeSegment`)
- `StretchSessionFrame.cycleStartMs → cycleStartSec`, `currentCycleMs → currentCycleSec`, `currentInhaleMs → currentInhaleSec`, `currentExhaleMs → currentExhaleSec`
- `getStretchFrame(segments, elapsedMs: number) → (segments, elapsedSec: number)`; body locals likewise rename
- **Exported `computeStretchTotalMs(settings): number | null → computeStretchTotalSec(settings): number | null`** (LSP rename so consumers update via find-references)
- Body of `buildStretchSegments` / `makeSegment` / `computeStretchTotalSec`: every `60_000 / bpm` literal becomes `60 / bpm`; every `* 60_000` (minutes-to-ms) becomes `* 60` (minutes-to-sec); the `MS_PER_MINUTE = 60_000` constant pattern (if any in this file) becomes `SEC_PER_MINUTE = 60`. The `CLAMP_EPSILON_MS = 1` constant inside `getStretchFrame` (representing 1ms) becomes `CLAMP_EPSILON_SEC = 0.001` (the same physical quantity in the new unit) — the clamp behavior is byte-identical.
