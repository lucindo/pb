<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → app | `loadPrefs()` reads the envelope; covered by `coerceCue` (Plan 01). |
| cross-tab `storage` / same-tab CustomEvent → hook | Event-driven re-reads cross a same-origin boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-03 | Spoofing | A page script firing a forged `hrv:prefs-changed` CustomEvent | accept | Same-origin only — any script that can dispatch the event already runs in the page and could call `savePrefs` directly. The hook re-reads `loadPrefs()` (whitelist-coerced by Plan 01's `coerceCue`); a forged event cannot inject an out-of-set cue value. No trust elevation. |
| T-25-04 | Tampering | The `storage` event payload | mitigate | The hook ignores the event payload entirely — it filters on `e.key === STATE_KEY` and then re-reads `loadPrefs()`, which re-coerces. The event is only a re-read trigger, never a data source. |

Client-side prefs sync only — no network, no auth, no server, no PII. Repudiation / Information Disclosure / DoS / Elevation-of-Privilege not applicable.
</threat_model>
