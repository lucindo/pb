## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Tampering | `coercePrefs` reading `cue` from a hand-edited / corrupt localStorage envelope | mitigate | `coerceCue` is a non-throwing whitelist coerce — any value not in `CUE_OPTIONS` falls back to `DEFAULT_CUE`. No `eval`, no dynamic key lookup; the existing object-shape guard (`!Array.isArray`, `typeof === 'object'`) already blocks prototype-pollution vectors. |
| T-25-02 | Denial of Service | A drifted `cue` value discarding the whole envelope and resetting all prefs | accept→mitigate | Per-field coercion isolates `cue` — a bad `cue` cannot corrupt theme/timbre/variant/locale. This is the explicit CONTEXT D-13 design. |

This plan is pure client-side data + types: no network, no auth, no server, no PII. Spoofing / Repudiation / Information Disclosure / Elevation-of-Privilege are not applicable.
</threat_model>
