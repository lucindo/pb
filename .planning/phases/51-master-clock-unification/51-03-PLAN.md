<objective>
Rebase NK session stats (`startedAtSec` capture + `elapsedSec` computation in `useNKEngine`) onto `audioCtx.currentTime` via the Phase 50 `SessionClock` seam (D-01 / D-02). This is a stats-only rebase per D-01: `useNKEngine`'s setTimeout-driven per-OM cadence is NOT changed; only the source of `clock.now()` shifts from a controller-local wall clock to the audio-backed proxy exposed by `useNaviKriyaAudio.clock`.

Practice-time semantics (D-07/D-08) extends to NK: when the AC suspends mid-NK-session (iOS lock), `clock.now()` freezes → `elapsedSec` freezes → recorded `elapsedMs` excludes the locked duration. Stats are now consistent across HRV / Stretch / Navi (success criterion #4 — "no regression across the three practices" with a consistent stats semantic).

NK has no in-session reconstruction path (D-12), so there is NO `onSessionClockReanchored` wiring on the NK side — the proxy's `setSource` is invoked only inside `begin()` (swap to audio clock) and `close()` / `closeAfterEndCue()` / unmount (revert to wall clock).

Purpose: Single-source-of-truth stats across all three practices. Closes diagnosis #1 + #2 for NK by construction (NK uses setTimeout not rAF for cadence, so the rAF-vs-AC divergence pattern from diagnosis #1/#2 already does not apply to NK cadence; consistent stats semantic is the remaining work).

Output:
- NK stats are AC-time-based — locked duration excluded from `recordNaviKriyaSession.elapsedMs`.
- `useNaviKriyaSessionController` no longer constructs a wall clock; consumes the audio-backed proxy from `useNaviKriyaAudio.clock`.
- Independent of Plan 51-02 — runs in parallel with HRV wiring after Plan 51-01.
</objective>
