<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host (HTTPS) → service worker | The SW intercepts every navigation/asset fetch under `/hrv/` scope; a poisoned cache would serve to all visits |
| deployed shell → installed app | An installed PWA keeps serving from cache across deploys until the SW updates |
| build tool → runtime bundle | `vite-plugin-pwa` generates `sw.js` + `manifest.webmanifest` that ship to users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-04 | Tampering | service-worker precache (`dist/sw.js`) serving stale or corrupted shell | mitigate | `cleanupOutdatedCaches: true` (D-04, Task 1) purges superseded precache entries on every SW activation; Workbox precaches only build-time, content-hashed assets — no dynamic/remote fetch is cached, so there is no runtime cache-poisoning vector. |
| T-27-05 | Tampering | stale shell served after a deploy (PWA-03) | mitigate | `registerType: 'autoUpdate'` sets `skipWaiting` + `clientsClaim`: a new SW activates and claims clients silently; combined with `cleanupOutdatedCaches`, the next app open serves the fresh shell. The current page intentionally keeps old JS until reload — no stale-shell-on-disk persists. |
| T-27-06 | Spoofing | Web App Manifest identity / scope mismatch | mitigate | `scope`/`start_url` auto-default to the Vite `base` `/hrv/`; the build-artifact verify in Task 1 asserts `scope === '/hrv/'`, so the SW cannot register at an over-broad `/` scope (Pitfall 3). Manifest is a static, in-repo, HTTPS-served artifact — no spoofing surface. |
| T-27-07 | Denial of Service | mid-session reload interrupting a running breathing session | mitigate | No `virtual:pwa-register` import exists in `src/` — the Task 1 verify greps `vite.config.ts` and forbids the string; the injected `registerSW.js` registers the SW without a `location.reload()` listener (D-03, PWA-03). An update can never interrupt a running session. |

**Residual risk:** LOW. The surface is static PWA assets on an HTTPS host with no backend, no auth, and no user input. No `high`-severity threats; all are mitigated by locked configuration values verified as build artifacts.
</threat_model>
