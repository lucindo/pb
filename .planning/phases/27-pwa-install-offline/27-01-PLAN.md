## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → local node_modules | Build-time dependency download crosses an untrusted-source boundary |
| static icon files → installed app identity | Icons become the app's home-screen identity at install time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Tampering | `package-lock.json` / npm dependency tree | mitigate | `npm install --save-dev` writes a locked, integrity-hashed resolution into `package-lock.json`; the lockfile is committed so the resolved tree is reproducible and reviewable. No transitive dep enters `dependencies`. |
| T-27-02 | Spoofing | static install icons (PWA identity) | accept | Icons are static build artifacts authored in-repo from a fixed SVG source; no dynamic/remote icon fetch. Misleading-identity risk is limited to the repo owner's own artwork — low residual risk, no PII or auth involved. |
| T-27-03 | Elevation of Privilege | vite-plugin-pwa as a runtime dependency | mitigate | Plugin is confined to `devDependencies`; it runs only at build time and ships no code into the runtime bundle. The `dependencies`-unchanged invariant check (Task 1 verify) enforces this. |
</threat_model>
