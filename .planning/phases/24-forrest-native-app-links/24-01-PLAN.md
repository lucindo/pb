<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app UI → external store sites | Two new outbound `<a>` links navigate the user from the trusted app surface to apps.apple.com and play.google.com (untrusted external origins). |
| Source content → rendered DOM | Static URL/label strings authored in `learnContent.ts` / `strings.ts` flow into `href`/text of `<a>` elements. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Tampering | New `<a target="_blank">` to external store sites — reverse tabnabbing (opened page gains `window.opener`) | mitigate | Both new `<a>` elements carry `rel="noopener noreferrer"` per D-04; LearnDialog.test.tsx asserts this on both new links and the existing `every <a> element` test covers them. |
| T-24-02 | Tampering | `href` value of the two new links — dangerous-scheme injection (`javascript:`, `data:`) | mitigate | URLs are static string literals committed in `learnContent.ts` (no user input, no interpolation); `learnContent.test.ts` dangerous-schemes `allUrls` check is extended to include both new URLs. |
| T-24-03 | Information disclosure | Referrer leakage to external store origins | accept (also mitigated) | `rel="noreferrer"` (part of `noopener noreferrer`) suppresses the `Referer` header; no PII is in the app URL regardless. |
| T-24-04 | Spoofing | Link labels could falsely imply Forrest Knutson authored/endorses the apps | mitigate | D-08 neutral framing — heading and labels name only the "Resonant Breathing" app, never assert Forrest's authorship/ownership (verified store developer is "John Goodstadt"); the `clinical-verbs` guard in `learnContent.test.ts` remains active for body copy. |

No `high` severity threats — phase scope is two static outbound links inside an existing dialog. All threats are `mitigate` or `accept`.
</threat_model>
