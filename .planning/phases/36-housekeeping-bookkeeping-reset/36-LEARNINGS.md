# Phase 36 Learnings: housekeeping-bookkeeping-reset

## Decisions

### D-01 — Single-batch restore → backfill → archive flow
`git restore .planning/phases/` upfront restored all 6 v1.5 phase dirs, then every artifact was backfilled in place, then a single `git mv .planning/phases/3{0..5}-* .planning/milestones/v1.5-phases/` archived them at the end. **Rationale:** Simpler to reason about than a per-phase loop, fewer state transitions, preserves rename history (vs the alternative of writing backfills directly to archive paths via `git show`).

**Source:** Execution dialog this session (orchestrator decision before Wave 1)

### Determinism is not idempotency
The original HOUSE-09 idempotency test ran `migrateEnvelope({version:1,…}, 1)` twice on identical fresh inputs and asserted `expect(once).toEqual(twice)`. That tests function determinism (same input → same output), not idempotency (running the operation on its own output is a no-op). A regression that mutated input or behaved differently on re-migration would still pass the original assertion.

**Context:** True idempotency test feeds the v3 output back through `migrateEnvelope(once, STATE_VERSION)` and asserts equality with the first pass. The v1→v2 block at storage.test.ts:180-195 has the correct pattern. Fix landed in commit f3ea0bc.
**Source:** 36-REVIEW.md IN-01, 36-REVIEW-FIX.md, fix commit f3ea0bc

## Patterns

### Backfill body opener convention
For all backfilled artifacts (VALIDATION.md, SECURITY.md, etc.):
```markdown
# Phase 12 — Validation Strategy

_Backfilled retroactively for Phase 12 (shipped 2026-05-11). Frontmatter `created: 2026-05-20` reflects backfill date; the audited code surface is the Phase 12 implementation present in `main`._

> Per-phase validation contract. …
```
Italic line immediately under the H1 title, before any existing blockquote-tagline.

**When to use:** Any retroactively-written validation/security/verification artifact where the audited code shipped earlier than the audit document.
**Source:** 36-PATTERNS.md "Backfill body opener", 36-CONTEXT.md D-03

### Inline ZERO_STATS_LITERAL for migration tests (no circular import)
Migration tests at `src/storage/storage.test.ts` MUST inline the zero-stats shape:
```typescript
const ZERO_STATS_LITERAL = {
  totalSessions: 0,
  totalElapsedSeconds: 0,
  lastSessionAtMs: null,
  lastSessionDurationSeconds: null,
}
```
Do NOT `import { ZERO_STATS } from './stats'` — `stats.ts` imports from `storage.ts`, so the import would create a circular dep. Same pattern as the v2→v3 block at lines 254-259 and the implementation at `storage.ts:112-128`.

**When to use:** Any new migration test or storage-ladder test in this codebase. The circular-import constraint is explicitly called out at `storage.ts:112-113`.
**Source:** 36-PATTERNS.md §1 "Inline ZERO-stats literal", 36-06-SUMMARY.md

### Single `git mv` for bulk archive moves preserves rename history
Plan 36-07 moved 106 files (6 v1.5 phase directories) with a single `git mv` command. The resulting commit shows up under `git log --diff-filter=R HEAD~1..HEAD` and rename relationships are preserved through `git log --follow`.

**When to use:** Any milestone archive move that relocates a directory tree. Avoid `git rm + git add` (loses rename history) or per-file `git mv` (clutters commit). One commit, one move.
**Source:** 36-07-SUMMARY.md (commit 55d057a)

### True idempotency assertion pattern
For migration / transform tests, true idempotency feeds the output back through the transformation:
```typescript
const once = migrateEnvelope({ version: 1, settings, stats }, 1)
const twice = migrateEnvelope(once, STATE_VERSION)
expect(twice).toEqual(once)
```
Not `migrateEnvelope(input, 1)` twice on fresh inputs — that's determinism, not idempotency.

**When to use:** Any test labelled "is idempotent" for a function. The v1→v2 analog at `storage.test.ts:180-195` is the in-repo reference. A regression that mutates input or treats v3-shaped input differently from v1-shaped input would slip past the determinism formulation.
**Source:** 36-REVIEW-FIX.md (IN-01 fix), fix commit f3ea0bc
