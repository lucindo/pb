<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| settings prop → SettingsForm steppers | Settings arrive from coerced storage; the form must not crash on edge values (e.g. empty targetBpm option set) |
| frame prop → SessionReadout | Frame is engine-produced; stretch fields are optional and may be undefined |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-08 | Tampering | targetBpm stepper options | mitigate | `STRETCH_INITIAL_BPM_OPTIONS` enforces `initialBpm >= 1.5` so `BPM_OPTIONS.filter(v => v < initialBpm)` always yields at least `[1.0]` — the stepper never receives an empty options array (RESEARCH Pitfall 4). The auto-correct handler keeps `targetBpm` in range when `initialBpm` drops. |
| T-22-09 | Tampering | SessionReadout stretch render | mitigate | The BPM/stage block renders only when `frame.currentBpm !== undefined`; a standard frame (undefined) takes the unchanged path. `frame.currentBpm.toFixed(1)` is reached only after the `!== undefined` guard. |
| T-22-10 | Information Disclosure | rendered BPM value | accept | The live BPM readout is an intentional UX feature (D-13, approved 22-UI-SPEC.md). It exposes only the user's own session rate — no PII, no cross-user data. Not a disclosure threat. |

No high-severity threats. The settings form consumes only discrete option-array
values; rendered output is derived from the user's own engine state.
