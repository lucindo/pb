## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage envelope → coerceSettings | Persisted JSON is attacker/corruption-controllable; stretch fields may be missing, drifted, or malformed |
| selectedSettings → startSession | Settings flow into segment-table construction at session start |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-04 | Tampering | coerceSettings stretch fields | mitigate | Per-field `isValid*` predicate gate; any drifted/missing field falls back to a known-good default. No field can carry an out-of-range value past the coercer. Forward-compat read means an absent field is indistinguishable from a corrupt one — both default. |
| T-22-05 | Tampering | startSession segment build | mitigate | `buildStretchSegments` consumes `lockedSettings` which is a clone of coerced settings; `createBreathingPlan` still calls `validateSettings` which throws on out-of-range values, so a malformed settings object cannot silently produce a degenerate segment table. |
| T-22-06 | Denial of Service | localStorage envelope size | accept | Six new scalar fields add negligible bytes to the envelope; localStorage quota (~5MB) is nowhere near threatened. No mitigation needed. |

No high-severity threats. STATE_KEY/STATE_VERSION unchanged — no schema-version downgrade surface introduced.
