<objective>
Extend `useSessionEngine`'s rAF tick with the per-tick elapsed-delta clamp (D-05/D-06), the sessionStartCtxTime rebase-on-clamp-fire (D-07), and the synchronous lastClockNow reset inside `reanchorSessionClock` (D-08). Closes SCHED-01 (catch-up burst suppression — diagnosis #4).

Purpose: A long hidden window (tab background, desktop screen lock without AC suspension) lets rAF pause but `audioCtx.currentTime` keeps running. The first rAF tick back sees a huge `clock.now() - lastClockNow` delta and — without a clamp — would compute a session elapsed that races forward by the hidden duration. The clamp caps the per-tick delta to 100ms; the rebase pushes `sessionStartCtxTime` forward so practice-time stays correct (hidden ≠ attention time, mirroring Phase 51's iOS lock semantics).

Output: A new `lastClockNowRef` declared at hook scope, a clamp + rebase block inserted inside the existing rAF tick body (inside the `setState((currentState) => ...)` updater per AH-WR-05), a synchronous reset inside `reanchorSessionClock`, and ≥6 Vitest behavioral tests in `useSessionEngine.test.tsx` that lock the contract.
