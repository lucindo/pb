| React component → previewContext module | Synchronous function call across two pure modules in the same trust domain. No serialization, no untrusted input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-08 | T (Tampering) | onClick handler call order | accept | The onClick handler is two synchronous calls (`setTimbre(id); playInhalePreview(id)`). Order is enforced by JS sequence; no race surface. TS narrowing on `id: TimbreId` prevents arbitrary string payloads. |
| T-40-09 | E (Elevation of privilege) | bypass of `disabled={inSessionView}` | accept | The browser refuses to fire click events on `<button disabled>`. To bypass, an attacker would need DOM-injected JS execution, which is already a higher-privilege escalation than this surface introduces. | T-40-10 | D (Denial of service) | spam-tap audio surface | accept | Same disposition as T-40-03 (Plan 01). cueSynth's per-call envelope + `ended` cleanup bounds the resource footprint; rapid tapping degrades gracefully. |
