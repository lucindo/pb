All values are CSS var tokens; no hardcoded hex in components (enforced by `theme.no-hardcoded-classes` guard). Tokens resolve to theme-specific hex at runtime.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-breathing-bg` | App background behind the dialog backdrop |
| Secondary (30%) | `--color-breathing-surface` | Dialog panel background (`bg-[var(--color-breathing-surface)]`) |
| Accent (10%) | `--color-breathing-accent` | Button borders, focus rings, banner border-top |
| Accent strong | `--color-breathing-accent-strong` | Section label text, button text, first iOS step highlight |
| Muted | `--color-breathing-muted` | Banner body text, iOS step 2 & 3 text |
| Modal backdrop | `--color-modal-backdrop` | Dialog backdrop overlay |
| Surface soft | `--color-breathing-bg-soft` | Button hover/active background state |
| On-accent | `--color-breathing-on-accent` | Not used in this phase (no filled accent button) |

Accent (`--color-breathing-accent`) reserved for:
- Border on picker and button elements inside SettingsDialog
- Focus ring on interactive elements (`focus-visible:ring-breathing-accent`)
- Top-border separator on InstallBanner (existing, unchanged)

Accent-strong (`--color-breathing-accent-strong`) reserved for:
- Install row section label `<p>` text
- Install button label text
- iOS steps toggle button text
- First step list item text (iOS steps block — matches existing InstallBanner pattern)

No destructive token: this phase introduces no destructive actions. The install trigger and iOS expand are non-destructive. The Close button (existing) is non-destructive.

## Install Row — Interaction Contract

The install row is the core new UI surface in this phase. All interaction rules below derive from CONTEXT.md decisions D-01 through D-10.

### Visibility gate

Show the install row when ALL of these are true:
- `installable === true` (i.e., `deferredPrompt !== null` OR `isIOS === true`)
- `isStandalone === false`

When hidden: no row, no placeholder, no "already installed" message (D-04). The row is simply absent.

### Render: Android / Desktop Chrome / Desktop Edge path (non-iOS)

```
[Section label: "Install for offline use"]
[Install button: "Install"]
```

- Section label: `<p className="text-sm font-semibold text-[var(--color-breathing-accent-strong)]">`
- Install button: `<button type="button" onClick={() => void onInstall()}>` with `min-h-[44px]`
- Button copy reuses `strings.install.installButton` (same as banner — "Install" / "Instalar")
- Button styling: matches Close button pattern — `rounded-full border border-[var(--color-breathing-accent)] bg-[var(--color-breathing-surface)] px-5 py-2 text-sm font-semibold text-[var(--color-breathing-accent-strong)] shadow-sm transition hover:bg-[var(--color-breathing-bg-soft)] active:bg-[var(--color-breathing-bg-soft)] motion-reduce:transition-none focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-breathing-accent focus-visible:ring-offset-2`

### Render: iOS path

```
[Section label: "Install for offline use"]
[Toggle button: "How to install"  ▶]
  [When expanded: numbered steps 1–2–3 + Share glyph]
```

- Section label: same as non-iOS path
- Toggle button: `<button aria-expanded={iosExpanded} aria-controls="settings-ios-steps">` — `strings.install.iosStepsButton`
- Steps: `<IosInstallSteps id="settings-ios-steps" strings={strings.install} />` rendered when `iosExpanded === true`
- Toggle button styling: `min-h-[44px] text-sm font-semibold text-[var(--color-breathing-accent-strong)] focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-breathing-accent focus-visible:ring-offset-2` — matching InstallBanner iosStepsButton pattern

### In-session behavior

When `inSessionView === true`: disable the install button (or the toggle). Matching the `disabled={inSessionView}` pattern applied to all 5 pickers. The dialog auto-closes on session start (WR-09) so this is belt-and-suspenders. Apply `disabled` attribute to the action button; apply `disabled:cursor-not-allowed disabled:opacity-45` consistent with picker buttons.

### Post-install behavior

After native install prompt completes: `isStandalone` flips to `true` on next dialog open. Install row hides on re-open. No in-session response to standalone flip needed.

## IosInstallSteps Component Contract

### Props

```typescript
interface IosInstallStepsProps {
  id: string                    // aria-controls target — MUST be unique per surface
  strings: Pick<UiStrings['install'], 'iosStep1' | 'iosStep2' | 'iosStep3'>
}
```

### Markup pattern

```
<div id={id} aria-live="polite" className="pt-4 text-sm leading-6">
  <ol className="list-decimal pl-5">
    <li className="text-[var(--color-breathing-accent-strong)]">
      {strings.iosStep1}{' '}<IOsShareIcon />
    </li>
    <li>{strings.iosStep2}</li>
    <li>{strings.iosStep3}</li>
  </ol>
</div>
```

- `IOsShareIcon` moves into `IosInstallSteps.tsx` alongside the component (no circular dependency risk)
- InstallBanner passes `id="install-ios-steps"`, SettingsDialog passes `id="settings-ios-steps"` — no ID collision

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| 44×44px touch targets | `min-h-[44px]` on all interactive elements (install button, iOS toggle) — same as banner |
| Focus ring | `focus-visible:ring-2 focus-visible:ring-breathing-accent focus-visible:ring-offset-2` — same as every picker |

| No new ARIA roles | Install row uses only existing patterns: `<p>` label + `<button>` action + `<ol>` steps |
