<objective>
Extend the App-side orchestrator hook in `src/hooks/useFeatureFlags.ts` so it (1) seeds a persisted-prefs snapshot from `loadPrefs()` at mount, (2) listens for cross-tab `storage` writes filtered on `STATE_KEY` (cross-tab parity per Phase 8 D-04a), (3) listens for same-tab `hrv:prefs-changed` CustomEvents filtered on the four new keys plus `undefined` (D-11), and (4) calls the new 2-arg `readFeatureFlags(search, persisted)` from Plan 01 to resolve the final `FeatureFlags` per-field.

The existing `popstate` subscription via `useSyncExternalStore` stays untouched — URL navigation continues to trigger a re-read; the `search` value is one input to the resolver and the persisted snapshot is the other.

Purpose: closes PREFS-01 (persisted prefs reach the app), PREFS-02 (query wins over persisted on mount), PREFS-03 (returning users with no persisted change see byte-identical defaults). The four downstream consumers — `PracticeScreen.tsx:64,73-75` reading `vm.featureFlags.{switcherIcon, breathingShape, orbIdle, ringCue}` and `useAppViewModel.ts:54,183` reading the hook return — are NOT touched in this plan; their interface (the `FeatureFlags` type) is unchanged.

Output: modified `useFeatureFlags.ts` with two new `useEffect` listener blocks + a `useState` for the persisted snapshot, plus extended `useFeatureFlags.test.ts` covering the new listeners.
