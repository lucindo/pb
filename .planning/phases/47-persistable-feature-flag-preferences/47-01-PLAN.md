<objective>
Extend the pure resolver in `src/featureFlags.ts` to a 2-argument signature `readFeatureFlags(search, persisted)` that merges query-string > persisted > default per-field. Add a new `readQueryFeatureFlagOrNull<T>` helper that returns `null` when the query value is absent OR unparseable, so the `??` chain at the resolver falls through to the persisted snapshot for both cases (D-07).

Export the four `*_FLAG` consts so the storage-layer plan (Plan 02) can import `.defaultValue` and `.parse` directly — single source of truth for production defaults and alias tables.

Purpose: closes PREFS-01 (resolver knows about persisted) and PREFS-02 (query override behaviour preserved verbatim for valid values; invalid values now fall through to persisted instead of masking it).

Output: modified `src/featureFlags.ts` + extended `src/featureFlags.test.ts` with new 4-way resolver test block. No production caller is updated in this plan — `useFeatureFlags.ts` (the only non-test caller per CONTEXT D-08) is updated in Plan 03 and will still compile after this plan because the new second parameter accepts a `FeatureFlags` and the existing call sites in tests are updated here.
</objective>
