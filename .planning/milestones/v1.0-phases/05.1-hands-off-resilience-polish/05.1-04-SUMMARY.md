# Phase 5.1 Plan 04: Safari `.orb` WR-03 Sizing Re-Fix Summary

**One-liner:** WR-03 explicit-positioning applied to `.orb` + outer ring in both render sites, closing the Safari Desktop orb-collapse-to-58% bug exposed by Plan 02 UAT.

## What Was Done

Plan 02 UAT (`05.1-UAT.md` Task 3) exposed a deeper root cause: on Safari Desktop,
`.orb` (which used `inset-0`) collapsed to 209 px instead of the expected 360 px.
Root cause: abs-pos children of a `display:grid; place-items:center` container are
sized against an implicit auto-track on Safari. The largest non-abs sibling (the inner
ring, sized to 58%) set the track size, causing `inset-0` to resolve to 58% of parent.

Plan 04 applies the WR-03 explicit-positioning pattern (already used by the inner ring)
to the two remaining abs-pos children in both render sites.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Inner-ring test assertions use floating-point-tolerant regex**
- **Found during:** Task 1 verification
- **Issue:** jsdom serializes `0.58 * 100` as `57.99999999999999%` not `58%`. The plan's
  test body specified `expect(style).toMatch(/width:\s*58%/)` which would NEVER go GREEN.
- **Fix:** Changed inner-ring width/height assertions in A7 and B7 to use `/width:\s*5[78](\.\d+)?%/`
  regex that accepts both `58%` and `57.99...%`.
- **Files modified:** `src/components/BreathingShape.test.tsx`
- **Commit:** 9ed80c3 (included in RED commit)
