## Deviations from Plan

**Two deviations, both UAT-driven, both committed as consolidated fixes per anti-pattern policy:**

### 1. D-34 stale-closure bug (fixed in `c510d49`)

- **Found during:** UAT cycle 2 (real iPhone)
- **Issue:** `useAudioCues.resume()` checked `audioStatus === 'needs-resume'` after `await engine.resume()` to decide reconstruction. `audioStatus` is captured by useCallback closure — within the same invocation, it stays stale even after a successful resume's statechange listener has set it to 'ok'. Every successful gesture resume falsely escalated to reconstruction.
- **Fix:** Added `engine.state` getter (live `audioCtx.state` read); hook reads `engine.state` instead of React state.
- **Note:** Cycle 3 proved that even with this fix, iOS doesn't actually recover audio. The fix is structurally correct and is retained, but its premise (plain resume works on iOS gesture) does not hold.

### 2. D-31/D-33/D-35 mechanism change + iOS gesture preservation + re-anchor offset (kitchen-sink fix `213c2c7`)

- **Found during:** UAT cycle 2 + cycle 3 (real iPhone)
- **Issues (three intertwined):**
  - Plain `engine.resume()` returns `state='running'` on iOS but AC.currentTime never advances and no audio plays. State machine lies.
  - Reconstruction was awaiting `oldEngine.close()` before constructing the new AC; iOS consumed gesture token at the first await, so the new AC ran outside gesture context.
  - Re-anchor formula set `audioAnchorRef = newAC.now()` (≈ 0) which made boundary formula evaluate to large future audioTime relative to new AC.
- **Fix (one commit, three changes):**
  - `useAudioCues.resume()` always reconstructs (skips `engine.resume()`).
  - `useAudioCues.reconstructEngine()` creates new AC FIRST (sync), closes old AFTER (fire-and-forget).
  - `App.tsx onAudioReanchorRequired` subtracts `session.currentFrame.elapsedMs / 1000` from the new audio anchor.
- **Note:** This fix is structurally correct and necessary; tests confirm 312/312 green. But cycle-3 UAT revealed a deeper iOS limitation (audio session unrecoverable mid-page) which this fix cannot address — that is the v1.x carry-forward.

**Total deviations:** 2 commits, 0 cascading hotfix loops. Honored `.continue-here.md` "Hotfixing in many small commits" anti-pattern by batching cycle-3 hypotheses into one kitchen-sink commit.

**Impact on plan:** D-31/D-33/D-35 spec wording preserved; D-31/D-33 mechanism documented as "always reconstruct" instead of "resume-first"; D-35 mechanism extended with session-elapsed offset. The locked decision IDs themselves are not invalidated, but they have new implementation notes.
