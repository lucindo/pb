Box-model analysis (the bug's root cause):
- `inset-0` = `top:0; right:0; bottom:0; left:0;` → element is exactly the size of the parent (100%×100%).
- `border-width: 1.5px` + default `box-sizing: border-box` (set globally by Tailwind preflight) → the 1.5px border is INSIDE the 100% box.
- The orb at `scale(MAX_SCALE = 1.0)` is also 100%×100%.
- Result on Safari: the orb's edge at scale(1.0) coincides with the OUTER edge of the orb container box, but the outer-ring's BORDER edge sits 1.5px INSIDE that box. Visible gap = 1.5px.
- Chromium happens to render the borders aligned to the outer pixel boundary, masking the same arithmetic — but Safari's rendering is the spec-correct one.

Fix (D-10 option 2 — discretion pick): replace `inset-0` with explicit `inset: -1.5px` on BOTH outer-ring spans. This shifts the entire border-box outward by 1.5px so the border's OUTER edge sits 1.5px beyond the container's 100% box, matching the orb's scale(1.0) edge on every browser.
