# Phase 5.1 / Plan 06 — Pattern Map (iOS Safari audio root-cause fix)

**Mapped:** 2026-05-10

**Analogs found:** 7 / 7 (all in-tree; no external pattern needed)

## Scope

Plan 06 is the empirical follow-up to Plan 01's silent-failure on real iOS Safari. The fix widens the recovery path (gesture-attached `resume()` + engine reconstruction) and surfaces a user-tappable affordance via the existing MuteToggle. All analogs live in the current tree — no new directories, no new external deps.

**Anchor invariants** (carried forward from Plans 01-05):
- D-02 — engine stays React-free / DOM-event-free (`document.*` / `window.*` only inside hooks).
- D-09 (relaxed by D-38) — silent-absorb posture preserved EXCEPT for `InvalidStateError`, which surfaces to `audioStatus`.
- D-06 (preserved) — re-anchor ONLY on reconstruction (D-35), never on plain resume.
- Pitfall 2 — gesture chain spans synchronous `await` calls only; no `setTimeout` / `.then(...)` between resume and reconstruct.

  destination = new FakeAudioNode()

  private _listeners = new Map<string, Set<EventListener>>()

**Plan 06 four-state matrix tests (D-41) — copy the dispatch shape from line 337-355; add `_simulateResumeReject` invocations**:

| Test | Setup | Expected |
|------|-------|----------|
| `'running' → resume succeeds → audioStatus stays 'ok'` | Start, dispatch visible | `audioStatus === 'ok'`, no needs-resume |
| `'suspended' → resume succeeds → audioStatus stays 'ok'` | Start, `_simulateSuspend()`, dispatch visible | `audioStatus === 'ok'` post-resume |
