<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage -> PersistedStats | Persisted stats are read back through `coerceStats` — untrusted (could be hand-edited / corrupted) |
| recordNaviKriyaSession args -> envelope | `elapsedMs` / `roundsCompleted` come from the engine; numeric, in-process |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-06 | Tampering | A corrupted or hand-edited `roundsCompleted` value in localStorage | mitigate | `coerceStats` validates `roundsCompleted` with `isFiniteNonNegativeInt`; an invalid value coerces to `undefined` (renders as 0), the rest of the record survives — non-throwing per-field coercion (ASVS V5). |
| T-31-07 | Tampering | `recordNaviKriyaSession` accidentally mutating the `resonant` stats slice | mitigate | The function spreads `practices` and replaces ONLY the `naviKriya` key; `practices.test.ts` asserts `resonant.stats` is unchanged after the call (NK-08 isolation guarantee). |
| T-31-08 | Repudiation | A non-finite/negative `elapsedMs` inflating stats | mitigate | Early-return guards reject non-finite/negative `elapsedMs`; sub-threshold incomplete sessions are not recorded — mirrors `recordResonantSession`. |
</threat_model>
