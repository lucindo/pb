<objective>
Rename the fourth cue timbre `chime → flute` across every user-facing and persisted surface, and add a storage coercion so a returning user with a persisted `timbre: 'chime'` preference lands on `'flute'`.

Purpose: `'chime'` is removed from the `TimbreId` union in this plan. Without an explicit legacy remap, a persisted `'chime'` would silently fall back to `'bowl'` (DEFAULT_TIMBRE) and the user would lose the fourth-slot preference they chose. The spike-008 winner ("Flute — soft attack") is a user-facing rename, not an internal swap.

Output: `TimbreId`/`TIMBRE_OPTIONS` carry `'flute'`; EN + PT-BR strings say "Flute"/PT-BR equivalent; `TimbrePicker` lists Flute; `coerceTimbre` remaps legacy `'chime'`; regression tests for the rename and the coercion.

This plan is the dependency root: it makes `TimbreId` contain `'flute'`, which Plan 02 needs before it can key `TIMBRE_PRESETS` with `flute`.
</objective>
