Replace the (now-renamed) `flute` timbre preset's placeholder values with the spike-008 winning Flute recipe, add an `attackSec` field to `TimbrePreset`, and give `cueSynth` an optional soft-attack envelope mode so the flute's ~0.13 s breath onset is real.

Spike 008 proved a strike-envelope flute sits too close to Sine ‚Äî the soft attack is the load-bearing, defining feature. This plan delivers the actual audio (AUDIO-01) and keeps every other cue byte-identical.

Output: `TIMBRE_PRESETS.flute` is the spike-008 preset; `TimbrePreset` has `attackSec`; `cueSynth.scheduleBowlCue` does a linear `0 ‚Üí peakGain` ramp over `attackSec` then the existing exp decay when `attackSec > 0`, and is byte-identical to today when `attackSec` is `0`/absent.
</objective>

Pattern reference (DO NOT EDIT nkCueSynth.ts behaviour) ‚Äî src/audio/nkCueSynth.ts
`buildNKToneNodes` already implements an optional envelope mode via a
`number | PadEnvelope` union: a `number` ‚áí strike (instant attack + exp decay),
a `PadEnvelope` ‚áí linear fade in/hold/out. The strike path is the default so
untouched callers stay byte-identical. cueSynth's soft-attack mode mirrors that
PRINCIPLE ‚Äî optional mode, strike is default ‚Äî but cueSynth uses a simpler
trigger: `preset.attackSec`. attackSec absent/`0` ‚áí today's strike; attackSec > 0
‚áí linear ramp `0 ‚Üí peakGain` over attackSec, THEN the existing exp decay.
