<objective>
Close the Phase 34 verification BLOCKER (REVIEW CR-01): `coerceStretchSettings` validates
`initialBpm` and `targetBpm` independently but never enforces the cross-field invariant
`targetBpm < initialBpm`. A persisted or migration-seeded stretch slice with
`targetBpm >= initialBpm` passes coercion unchanged and reaches `buildStretchSegments`,
which silently collapses the ramp to a single inverted segment — the session accelerates
instead of decelerating. `validateStretchSettings` already encodes the invariant but is
dead code on the stretch start path.

Purpose: STRETCH-03 — Stretch is a distinct practice with its own settings; a corrupt
persisted slice must never reach the ramp engine. This is the sole open verification
blocker for Phase 34.
Output: `coerceStretchSettings` enforcing the cross-field invariant and coercing
`initialBpm` against `STRETCH_INITIAL_BPM_OPTIONS`, a defensive `RangeError` guard in
`buildStretchSegments`, regression tests for both, and a green verification run.
</objective>
