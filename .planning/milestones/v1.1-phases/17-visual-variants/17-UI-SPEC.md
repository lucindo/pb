# Phase 17 — UI Design Contract

> Visual and interaction contract for Phase 17 (Visual Variants). Generated by gsd-ui-researcher; checker upgrades `status: draft → approved`.

**Phase scope.** Three render variants (Orb default, Square, Ring) selectable from `VariantPicker` inside `SettingsDialog`. Picker mirrors `ThemePicker` posture verbatim: radiogroup with inline shape swatches, `{ disabled: boolean }` prop, self-owned `useVariantChoice` write path, `'hrv:prefs-changed'` CustomEvent with `detail.key = 'variant'`. Variant captured at session start via `sessionVariantRef` snapshot in `App.tsx`. Reduced-motion fixed-mid-scale + crossfade carries across all variants. CSS rename `.orb-ring--outer/--inner` → `.shape-marker--outer/--inner` (D-15). Render-local `data-variant` attribute on shape root only (D-16). No new color tokens — `--color-orb-in/out-*` reused across all variants (D-13).

**Pre-populated from upstream.** 21 of 25 design contract fields below are settled by CONTEXT.md (D-01..D-24) + RESEARCH.md atlas + carried-forward v1.0/v1.0.1/v1.1 token surface. The 4 remaining fields (Square border-radius, Ring inner-radius, Ring marker border-width, swatch primitive split) are locked here at the RESEARCH-recommended starting values; planner may revise after dev-server smoke without re-opening UI-SPEC.

## Spacing Scale

Multiples of 4 (Tailwind v4 default scale). Phase 17 uses a strict subset:

| Token | Value | Usage in Phase 17 |
|-------|-------|-------------------|
| sm (`gap-2`, `py-2`) | 8px | Gap between picker option buttons; vertical padding inside each button; gap between swatch + label |
| md (`px-3`, `mt-2`) | 12px / 8px | Horizontal padding inside picker buttons; top margin between section label and radiogroup |
| lg (`min-h-12`) | 48px | Minimum hit-target height on every picker button — satisfies 44×44 floor (VARIANT-06 + D-24) |
| Marker offset | -1.5px | Outer marker inset (`inset: -1.5px`) preserved verbatim from existing OrbBody render — counters Safari border-box gap at peak inhale (Phase 5.1 D-12 carry-forward) |
| Swatch size | 24px (`w-6 h-6`) | Inline shape swatch dimensions — matches ThemePicker icon scale convention |

**Exceptions:**
- `-1.5px` outer-marker inset is a sub-pixel anchor required by Safari rendering math (Phase 5.1 D-12); intentional non-multiple-of-4 value. Mirrored verbatim in Orb/Square/Ring marker render sites.
- 24px swatch size (vs the canonical 32px) chosen so the swatch + label stack fits inside `min-h-12 + py-2 + px-3` without overflowing the `grid-cols-3 gap-2` row layout at narrow viewport widths (dialog max-w-md).

Hybrid primitive strategy — RESEARCH-approved approach addressing Pitfall 8 (Ring swatch invisible at 24px under Option D radial-gradient):

## Interaction Contract

### Radiogroup keyboard navigation (mirror of ThemePicker)

| Action | Behavior | Source |
|--------|----------|--------|
| `Tab` from previous focusable element | Focus lands on the **selected** option button (or the first option if none selected) | Native `<button>` focus behavior in browser tab order |
| `Arrow Right` / `Arrow Down` | Move focus to next option button (wraps from last → first) | Native `<button>` does NOT implement radiogroup arrow-key navigation; defer to browser default focus on Tab. **Phase 17 does not implement arrow-key navigation explicitly** — same posture as ThemePicker (ThemePicker.tsx renders 6 buttons in a `grid grid-cols-3` with no arrow-key handler). Tab + Space/Enter to select. Documented as intentional limitation for v1.1. |
| `Space` / `Enter` | Activates focused button: `onClick={() => setVariant(id)}` fires | Native `<button>` activation |
| `Esc` (inside SettingsDialog) | Closes the dialog (NOT specific to the picker — owned by SettingsDialog's `<dialog cancel>` listener) | SettingsDialog.tsx:54-62 |

**Click handler:** `onClick={() => { setVariant(id) }}` — fires `useVariantChoice().setVariant(id)`, which:
1. Reads current prefs via `loadPrefs()` (fresh read — no stale closure).
2. Writes merged prefs via `savePrefs({ ...current, variant: id })`.
3. Updates local React state (`setVariantState(id)`) for optimistic-UI.
4. Dispatches `new CustomEvent('hrv:prefs-changed', { detail: { key: 'variant', value: id } })` (D-22 — same event name as theme; different `detail.key`).

**Cross-tab sync:** App-side `useVisualVariant` listens to (a) `'storage'` events filtered on `STATE_KEY`, and (b) `'hrv:prefs-changed'` events filtered on `detail.key === 'variant' || detail.key === undefined`. Both paths re-read `loadPrefs().variant` and update `liveVariant`.

**Mid-session protection (VARIANT-03):** Picker is `disabled={inSessionView}` — same-tab swap is impossible via UI. Cross-tab swap (via direct `localStorage.setItem(STATE_KEY, ...)` from DevTools or another tab) is intercepted by the `sessionVariantRef` snapshot mechanism in App.tsx: the `variant` prop passed to `BreathingShape` is `sessionVariantRef.current ?? liveVariant`. While `sessionVariantRef.current` is non-null (i.e., during an active session), `liveVariant` is ignored. Ref is set in `startSession` (App.tsx:315 area) before lead-in begins; cleared in the leave-running cleanup effect (App.tsx:476 area).

**Instant.** No transition on swatch shape or label position when a button is clicked. The selected-state border/fill swap uses the default Tailwind `transition` utility (150ms on `bg-color` + `border-color` only) — no swatch geometry tween. Matches ThemePicker behavior exactly.

| Open SettingsDialog (in-session) | Dialog visible; VariantPicker shows `aria-disabled=true` on radiogroup + `disabled=true` on each button + `opacity-45 cursor-not-allowed` styling | none | none |
| Click an option button (in-session) | No-op — `disabled=true` blocks click event from firing | none | none |
| Cross-tab `localStorage` write (in-session) | `liveVariant` updates inside `useVisualVariant`; rendered shape DOES NOT change (sessionVariantRef snapshot wins) | already done by other tab | already dispatched by other tab |
| Cross-tab `localStorage` write (idle) | `liveVariant` updates; if dialog is open, the picker selected-state updates to the new value (no re-mount) | already done by other tab | already dispatched by other tab |
| Session start (`onStartClick`) | `sessionVariantRef.current = liveVariant` is set BEFORE lead-in begins | none | none |
| Session end (leave-running cleanup effect) | `sessionVariantRef.current = null` | none | none |

## Responsive Behavior

| Viewport | Behavior |
|----------|----------|
| Mobile narrow (≤ 360 px) | Dialog sits at `max-w-md` (~448 px) but is constrained by viewport width; picker buttons remain in `grid grid-cols-3 gap-2` row. Each button stays ≥ 44 px wide (the `gap-2 = 8 px` × 2 + ~22 px option text fits comfortably at 320 px viewport: (320 − 48 px dialog padding × 2 − 16 px gap × 2) / 3 ≈ 75 px per button). |

| Tablet + desktop (≥ 640 px) | Phase label upgrades from `text-5xl` → `sm:text-6xl` (60 px); lead-in digit upgrades from `text-7xl` → `sm:text-8xl` (96 px). Picker buttons stay at `text-sm` (no responsive scale — picker is dialog-internal chrome). |

**No grid breakpoints.** `grid-cols-3` is constant across all viewports — 3 variant options always fit in a single row at every supported viewport width.
