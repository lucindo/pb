<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User pointer/keyboard → React onClick | Standard UI input; no untrusted data crosses |
| React component props ← App.tsx state | Internal boundary; `disabled` derived from `inSessionView`, `onClick` from useCallback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | T (Tampering) | `onClick={disabled ? undefined : onClick}` gating | mitigate | Defence-in-depth pair: JSX-layer handler removal (this plan) + App.tsx open-guard `if (inSessionView) return` (Plan 04). Both layers must agree before dialog opens during session. |
| T-15-02 | I (Information Disclosure) | aria-label disabled string | accept | "Settings (unavailable during session)" reveals app state to screen-reader users; intentional per WCAG (D-08 disable-not-hide). No PII; not a vulnerability. |
| T-15-03 | E (Elevation of Privilege) | aria-disabled vs HTML disabled | mitigate | Anchor uses `aria-disabled` (Landmine 4); button stays in tab order with no handler. Adversary attempting to fire onClick via dispatchEvent finds no handler attached — JSX-layer no-op holds. |
| T-15-04 | S (Spoofing) | Gear SVG vs LearnAnchor book SVG | accept | Decorative inline SVG; no clickjacking risk; no remote asset; `aria-hidden="true"` so screen-readers ignore icon and rely on aria-label. |
</threat_model>
