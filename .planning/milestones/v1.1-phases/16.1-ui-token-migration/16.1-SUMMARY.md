# Phase 16.1: UI Token Migration — Phase Summary

## Summary

Phase 16.1 closes the theme-propagation gap surfaced during Phase 16 human-verify: 16 production `.tsx` files plus `src/index.css` referenced hardcoded `text-slate-*` / `bg-teal-*` / `border-teal-*` / `text-white` / `bg-white` Tailwind classes that did not rebind on `data-theme` swap. After Phase 16.1, the entire production surface consumes `--color-breathing-*` CSS custom properties: dialogs, pickers, anchors, stepper, mute toggle, page chrome, breathing surface, readout, and the primary Start/Stop action button all rebind per palette. One new chrome token (`--color-breathing-on-accent`, D-01) was added across all 5 palettes (Light/Dark/Moss/Slate/Dusk) to provide a contrast-tuned foreground for accent-strong backgrounds (the primary button, body::selection, and destructive End/Reset buttons). A regression-guard Vitest test (`theme.no-hardcoded-classes.test.ts`, D-04) scans `src/**/*.tsx` for 10 banned patterns on every commit and is now GREEN — any future Phase 17/18/19 re-introduction of a banned class will fail CI. The THEME-05 contrast guard was extended with a 6th `it()` block per palette (accent-strong vs on-accent ≥ 1.5 luminance), and the existing 5 reduced-motion crossfade contrast iterations remain intact. The D-02 alpha-modifier strategy (Path A — Tailwind v4 `bg-[var(--token)]/N` form) was validated and applied across all 5 alpha sites. Per-commit green-gate held throughout (`tsc + eslint + build + test` exit 0 at every commit boundary). Zero new npm dependencies. Per-palette UAT (plan 06) approved after inline F2+F3 remediation; 5 palette-aesthetic findings (F1/F4/F5/F6/F7) deferred to Phase 16.2 (palette retune — out of 16.1 scope per CONTEXT.md).
