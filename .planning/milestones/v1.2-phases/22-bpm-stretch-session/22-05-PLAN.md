<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frame → audio boundary effect | The frame drives audio-clock scheduling; a malformed cycleStartMs would mis-schedule cues |
| audio scheduling → Web Audio clock | Cue times are committed to the audio thread; a wrong offset is audible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-11 | Tampering | boundary audio-time computation | mitigate | `cycleStartMs` is produced by `getStretchFrame` from a deterministic segment table built from `validateSettings`-checked settings; it cannot be arbitrarily injected. The `clampedAudioTime = Math.max(audioTime, liveAudioNow + SAFE_LEAD_SEC)` clamp (unchanged) bounds a too-early cue; the audio engine's callee-side clamp is the safety net for a too-late one. |
| T-22-12 | Denial of Service | per-cycle scheduling load | accept | One cue scheduled per phase boundary, same rate as a standard session (the ramp does not increase cue frequency). Segment table is small and built once. No mitigation needed. |
| T-22-13 | Repudiation | n/a | accept | No audit/logging requirement for a local breathing app; no user accounts. Out of scope. |

No high-severity threats. The change generalizes an existing deterministic
formula; no new input source or trust boundary is introduced.
</threat_model>
