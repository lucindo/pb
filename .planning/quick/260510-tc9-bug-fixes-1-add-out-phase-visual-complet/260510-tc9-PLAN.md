<objective>
Fix two user-reported UX defects in the HRV breathing webapp (v1.0 post Phase 5.1):

1. **Visual cue parity** — The Out-phase orb shrink has no visual reference comparable to the In-phase outer ring. Root cause: the inner reference ring already exists in `BreathingShape.tsx` but `theme.css` renders it with a thinner border (1 px vs 1.5 px) and lower-contrast color (alpha 0.45 vs 0.6), so on a smaller circumference it reads as a weak background detail rather than a symmetric cue.

2. **Bowl cue too short at low BPM** — The In/Out bowl cue is perceptually silent after ~3 s even though oscillators run ~7–9 s, because `setTargetAtTime` is exponential decay (`τ = 1.4/1.8 s` → audible ~3×τ ≈ 4–5 s). At BPM ≤ 3.5, each phase is ≥ 5 s and the user hears silence before the next boundary. Fix by stretching `decayTimeConstant` per phase duration (Direction A from the bug brief) so audible decay tracks the phase length.

Purpose: restore symmetric In/Out perceptual cues (visual + audio) so the app remains hands-off at low BPMs.
Output: One minor CSS edit + a small synth/engine signature extension threaded through useAudioCues + App, plus updated tests.
</objective>
